<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool dispatch | Metadata determines whether a tool is allowed in the current execution mode |
| approval gate | ApprovalPolicy determines whether tool execution requires user confirmation |
| abort signal | CancellationToken crosses thread boundary from HTTP handler to agent turn loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | ToolMetadata for MCP/dynamic tools | mitigate | lookupMetadata returns nullable; callers use conservative default (.mutating=true, .background_safe=false) for null |
| T-01-02 | Tampering | ExecutionMode set by LLM output | mitigate | ExecutionMode is set only by slash command or session config, never parsed from LLM response content |
| T-01-03 | Elevation of Privilege | ApprovalPolicy bypass via full autonomy | accept | full autonomy is set at session creation from operator config, not changeable by agent at runtime |
| T-01-04 | Denial of Service | CancellationToken use-after-free | mitigate | Token owned by session struct; threads joined before session deinit (existing SubagentManager pattern) |
</threat_model>
