## Decisions Made

- `extractTextAndPlan` returns slices into the original response (zero allocation). When both before-plan and after-plan text are present, `text` returns the before portion (plan blocks typically appear at the start). This is sufficient for the narration use case where the text is displayed before execution begins.
- `parseTaskPlan` receives only the inner XML content (between tags), not the full `<task_plan>` wrapper. This separates concerns: `extractTextAndPlan` finds the block boundary, `parseTaskPlan` decodes the content.
- `emitStepEvent` is a pure emit function — it does not advance the step or mutate plan state. This makes the narration side-effect explicit at call sites in the turn loop.
- `buildTaskDecompositionSection` is inserted between turn classification and safety, maintaining the section ordering principle from Plan 01 where classification precedes rules.

### T-1.5-05 Mitigation Verified

- Bounded scan: `std.mem.indexOfPos` + `std.mem.indexOf` — same pattern as `dispatcher.zig`
- Malformed input returns `null` via early return at every tag-not-found path

- Step count bounded by response length (no independent limit needed — Zig slice length is machine-word bounded)
