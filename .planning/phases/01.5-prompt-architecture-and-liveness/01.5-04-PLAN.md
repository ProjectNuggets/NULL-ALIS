    ```zig
    //! Learning loop — detects corrections and preferences in user messages,
    //! stores them as durable behavioral facts under durable_fact/behavior/ keys.
    //!
    //! Behavioral facts are distinct from raw memory recall. They are injected
    //! with higher priority during memory enrichment and change how the agent
    //! behaves, not just what it knows.
    //!
    //! Session scoping: per-user behavioral facts require session_id.
    //! Global workspace preferences (operator-set) use null session_id.

    const std = @import("std");

    pub const MAX_FACTS_PER_SESSION: usize = 100;

    pub const LearningSignal = enum {
        explicit_correction,   // "no, actually" / "that's wrong" / "I meant"
        explicit_preference,   // "always do X" / "prefer Y" / "never do Z"
        implicit_correction,   // "I meant the other one" / repeats request differently
    };

    pub const LearnedFact = struct {
        key: []const u8,
        content: []const u8,
        signal: LearningSignal,
    };
    ```

    Implement these functions:

    **`pub fn detectLearningSignals(allocator, user_message) ![]LearningSignal`**

    Heuristic string matching on the user message. Scan for patterns:
    - explicit_correction: contains "no, actually", "that's wrong", "that is wrong", "not what I", "I didn't mean", "I meant", "wrong,", "incorrect"
    - explicit_preference: contains "always ", "never ", "prefer ", "don't ever ", "from now on", "going forward", "remember that I", "keep in mind"
    - implicit_correction: contains "I meant", "what I really want", "let me rephrase", "try again"

    Pattern matching is case-insensitive. Use `std.ascii.lowerString` or iterate with `std.ascii.toLower`. Return a deduplicated list of detected signals. If no patterns match, return empty slice.

    **`pub fn factKey(allocator, fact_content) ![]const u8`**

    Generate a deterministic key for a behavioral fact:
    1. Lowercase the fact_content
    2. Hash with `std.hash.Fnv1a_64`
    3. Format as `durable_fact/behavior/{x:0>16}` (16-char hex of the hash)

    **`pub fn formatFactForEnrichment(fact_content) []const u8`**

    Returns the fact content as-is (behavioral facts are already stored as human-readable instructions). This is a trivial pass-through but exists as a named function for API consistency and future formatting.

    **`pub fn extractFactFromMessage(allocator, user_message, signals) !?[]const u8`**

    Given a user message with detected signals, extract the behavioral instruction:
    - For explicit_preference: take the full message as the fact (e.g., "always respond in English" -> "Always respond in English")
    - For explicit_correction: take the full message (corrections are context-dependent; the full message provides context)
    - For implicit_correction: return null (implicit corrections need more context than a single message)

    Allocate and return a copy of the extracted fact, or null if no extractable fact.

    Add inline tests at the bottom covering all behavior items.
