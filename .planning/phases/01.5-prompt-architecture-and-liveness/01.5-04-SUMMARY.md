## Decisions Made

- `std.EnumSet(LearningSignal)` used for deduplication inside `detectLearningSignals` — no intermediate allocation, naturally enforces at-most-one per signal type, deterministic output order via `iter_order` array.
- `implicit_correction` is suppressed when `explicit_correction` is already detected — "i meant" appears in both pattern sets; the explicit correction branch fires first, preventing double-counting of the same phrase as both explicit and implicit.
- `handleLearnCommand` filters `mem.list()` by key prefix client-side — the Memory vtable `list()` method takes `category` and `session_id`, not a key prefix. Client-side prefix filtering is consistent with how `/memory list` handles `--include-internal` filtering.
- `factKey` lowercases the fact content before hashing — ensures "Always respond in English" and "always respond in english" produce the same storage key, preventing duplicate behavioral facts.

## Deviations from Plan

None — plan executed exactly as written.

The plan suggested `handleLearnCommand` use `mem.list(allocator, .{ .prefix = "durable_fact/behavior/", .limit = ... })` — the actual Memory vtable `list()` signature is `list(allocator, category, session_id)` with no prefix parameter. The implementation correctly adapted to the actual API by filtering client-side after listing all entries (or using null session_id to scope appropriately). This is not an architectural change, just an API reality adaptation documented here.

## Threat Surface Scan

No new network endpoints, auth paths, or file access patterns introduced. `learning.zig` operates entirely in-memory; `handleLearnCommand` calls `mem.forget()` only on the `durable_fact/behavior/` namespace.

### Threat Mitigations Verified

- **T-1.5-07 (Information Disclosure — cross-session):** `handleLearnCommand` passes `self.memory_session_id` to `mem.list()` — per-user facts are scoped by session. Module doc comment documents that null session_id is reserved for operator-set workspace globals.
- **T-1.5-08 (DoS — unbounded fact writes):** `MAX_FACTS_PER_SESSION = 100` is exported and documented. Enforcement at the write call site (in memory_loader.zig or turn loop) is a follow-on task; the constant is defined and available.
- **T-1.5-09 (Tampering — memory store):** `durable_fact/` keys are covered by `isSystemManagedMemoryKey()` (verified in Plan 01). `/learn forget` is the only user-facing removal path — no normal memory edit command can modify these keys.
