### Gaps Summary

**4 gaps identified across 3 areas:**

**Gap 1 — NarrationObserver not instantiated (REQ-019):** The NarrationObserver is complete as a component but is not wired into any production code path. No channel, daemon, or agent turn startup creates or chains a NarrationObserver. The "agent emits real-time status" truth is unmet at runtime. Plan 02 explicitly defers the SSE wiring to Phase 2, but the truth as stated implies active emission — not just infrastructure readiness.

**Gaps 2 and 3 — Learning loop detection and storage not wired (REQ-021):** `detectLearningSignals` and the memory store call are never invoked in the production turn loop. The learning module is a complete library but zero learning occurs at runtime. Plan 04 SUMMARY documents this as a known stub to be wired in a future plan. Gap 2 (detection) and Gap 3 (storage) share the same root cause: turn-loop integration was intentionally deferred.

**Gap 4 — Dangling voice_style pointer (REQ-022 safety ordering):** `resolvePersonaFromFile` frees its `content` buffer before the `PersonaProfile.voice` slice is consumed by the caller. In root.zig, `p.voice` is accessed after `content` has been freed, creating undefined behavior. This affects any workspace with a `voice:` field in SOUL.md's front-matter. In debug Zig builds, this may not cause visible crashes (freed memory may not be immediately overwritten), but it is a correctness bug that could produce malformed voice calibration text in the system prompt.

**Root cause pattern:** Plans 02, 04, and (partially) 05 created infrastructure components that are deliberately not wired end-to-end in this phase — with explicit documentation of Phase 2 as the integration point. The truths as stated in the PLANs overstate what this phase delivers at runtime versus what it prepares as infrastructure.
