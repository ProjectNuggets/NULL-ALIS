   State transition validation in a helper:
   ```zig
   fn validateTransition(from: TaskStatus, to: TaskStatus) !void {
       const valid = switch (to) {
           .running => from == .queued,
           .succeeded, .failed, .timed_out => from == .running,
           .cancelled => from == .queued or from == .running,
           .lost => from == .running,
           .queued => false, // cannot transition back to queued
       };
       if (!valid) return error.InvalidTransition;
   }
   ```

   Task ID generation: use a monotonic counter formatted as zero-padded hex (e.g., "task_00000001"). The counter starts at 1 and increments. IDs are formatted into the 16-byte buffer on TaskEntry.

   sweepLost: iterate all entries, mark any with status == .running and (now_ms - updated_at_ms > LOST_TIMEOUT_MS) as .lost. Return count of swept tasks.

   MAX_TASKS enforcement: createTask returns error.LedgerFull when entries.count() >= MAX_TASKS.
