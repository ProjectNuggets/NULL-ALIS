## Decisions Made
- **Health command uses health.zig snapshot:** The Agent struct does not have access to ChannelManager, so the /health slash command shows component-level health from the global health registry. Per-channel detail (via ChannelManager) is available at the /api/v1/channels/health API endpoint which has access to gateway state and config.
- **Split /voice from /tts:** Previously both `/voice` and `/tts` mapped to `handleTtsCommand`. Now `/voice [on|off]` is a simple toggle (sets tts_mode=always, tts_audio=true), while `/tts` retains the full TTS configuration interface (provider, limit, summary, etc.).
- **Security review uses policy fields from agent:** The `handleSecurityReviewCommand` extracts `workspace_only` and `max_actions_per_hour` from `self.policy` (SecurityPolicy) when available, with safe defaults. The API endpoint reads directly from `config_opt` for full SecurityConfig access.
- **Both API endpoints are GET-only:** Return 405 for non-GET methods. Auth is provided by `validateInternalServiceToken` which runs at the top of `handleApiRoute`.
