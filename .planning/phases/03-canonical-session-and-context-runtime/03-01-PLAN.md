Create `src/session/identity.zig` with these types and functions:

1. **SessionLane enum** with 4 variants matching the existing convention:
   ```zig
   pub const SessionLane = enum {
       main,
       thread,
       task,
       cron,
   
       pub fn toSlice(self: SessionLane) []const u8 {
           return switch (self) {
               .main => "main",
               .thread => "thread",
               .task => "task",
               .cron => "cron",
           };
       }
   
       pub fn fromSlice(s: []const u8) ?SessionLane {
           if (std.mem.eql(u8, s, "main")) return .main;
           if (std.mem.eql(u8, s, "thread")) return .thread;
           if (std.mem.eql(u8, s, "task")) return .task;
           if (std.mem.eql(u8, s, "cron")) return .cron;
           return null;
       }
   };
   ```

2. **SessionIdentity struct** — the canonical identity for any session:
   ```zig
   pub const SessionIdentity = struct {
       user_id: []const u8,
       lane: SessionLane,
       lane_id: ?[]const u8,  // null for main, thread_id/task_id/job_id for others
       session_key: []const u8,  // the full canonical key string
   
       pub const PREFIX = "agent:zaki-bot:user:";
       pub const MAX_KEY_LEN = 255;
   };
   ```

3. **parseSessionKey** — parse a canonical session key string into its parts. Pattern: `agent:zaki-bot:user:{user_id}:{lane}` or `agent:zaki-bot:user:{user_id}:{lane}:{lane_id}`. Returns `ParseError` union:
   ```zig
   pub const ParseError = error{
       InvalidPrefix,
       MissingUserId,
       EmptyUserId,
       InvalidLane,
       MissingLaneId,
       KeyTooLong,
   };
   
   pub fn parseSessionKey(session_key: []const u8) ParseError!SessionIdentity
   ```
   Implementation:
   - Reject if len > MAX_KEY_LEN or len == 0
   - Check starts with `SessionIdentity.PREFIX` ("agent:zaki-bot:user:"), return InvalidPrefix if not
   - Extract rest after prefix, split on first `:` to get user_id
   - If no `:` found, return MissingUserId (no lane segment)
   - If user_id segment is empty, return EmptyUserId
   - Split remaining on first `:` to get lane name and optional lane_id
   - Parse lane via SessionLane.fromSlice, return InvalidLane if null
   - For thread/task/cron lanes, require lane_id (non-empty after the lane name `:`) — return MissingLaneId if absent
   - For main lane, lane_id is null (ignore any trailing content)
   - Return SessionIdentity with all fields pointing into the input slice (zero-allocation)

4. **formatSessionKey** — format a session key from parts into a caller-provided buffer:
   ```zig
   pub fn formatSessionKey(buf: []u8, user_id: []const u8, lane: SessionLane, lane_id: ?[]const u8) error{BufferTooSmall}![]const u8
   ```
   Format: `agent:zaki-bot:user:{user_id}:{lane}` for main, `agent:zaki-bot:user:{user_id}:{lane}:{lane_id}` for others. Use `std.fmt.bufPrint`.

5. **isOwnedBy** — check if a session key belongs to a user (replaces gateway.zig's sessionKeyOwnedByUser):
   ```zig
   pub fn isOwnedBy(session_key: []const u8, user_id: []const u8) bool
   ```
   Check: starts with `"agent:zaki-bot:user:{user_id}:"`.

6. **Inline tests** (minimum 12 tests):
   - Parse valid main key: `"agent:zaki-bot:user:42:main"` -> user_id="42", lane=.main, lane_id=null
   - Parse valid thread key: `"agent:zaki-bot:user:42:thread:conv-2"` -> lane=.thread, lane_id="conv-2"
   - Parse valid task key: `"agent:zaki-bot:user:42:task:t-99"` -> lane=.task, lane_id="t-99"
   - Parse valid cron key: `"agent:zaki-bot:user:42:cron:job-7"` -> lane=.cron, lane_id="job-7"
   - Parse invalid prefix returns InvalidPrefix
   - Parse empty user_id returns EmptyUserId
   - Parse unknown lane returns InvalidLane
   - Parse thread without lane_id returns MissingLaneId
   - Parse key > 255 chars returns KeyTooLong
   - Format main key matches expected string
   - Format thread key matches expected string
   - isOwnedBy returns true for matching user, false for non-matching
   - SessionLane.fromSlice round-trips with toSlice for all 4 variants

Import only `const std = @import("std");` — no external dependencies.
