## Decisions Made

- NarrationFrameType lives in `observability.zig` alongside `ObserverEvent`. If it lived in `narration.zig`, adding `narration_frame` to `ObserverEvent` would create a circular import. Both files import each other — Zig disallows this.
- `emitFrame` calls both the callback (if registered) AND re-emits as a `narration_frame` event through the inner observer. The callback serves immediate synchronous consumers (e.g., a CLI progress printer); the event bus emission serves future SSE/channel consumers in Phase 2 without requiring direct coupling.
- Safety comment placed at the first production `self.history.append` call (user message append) so any engineer reading the turn loop sees the constraint immediately.

## Deviations from Plan

None — plan executed exactly as written.

The only minor adaptation: the plan suggested `FrameType` might need to be defined at narration.zig module level and referenced from observability.zig via `@import`. Instead it was defined directly in `observability.zig` as `NarrationFrameType` and re-aliased in narration.zig as `pub const FrameType = observability.NarrationFrameType`. This avoids the circular import without any architectural change. The plan explicitly anticipated this approach as the preferred option.
