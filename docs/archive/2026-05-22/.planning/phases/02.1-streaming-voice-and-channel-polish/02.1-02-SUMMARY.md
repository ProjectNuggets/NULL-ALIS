## Key Artifacts

### src/channel_health.zig
- `ChannelHealthEntry` struct: name, listener_type, state, failure_count, backoff_ms, last_error, component_status
- `OverallStatus` enum with `fromChannels()`: any gave_up => unhealthy, any restarting => degraded, else healthy
- `aggregateChannelHealth()`: iterates ChannelManager entries, queries health registry for component status
- `formatHealthJson()`: `{"status":"healthy","uptime_seconds":3600,"channels":[...]}`
- `formatHealthText()`: human-readable with per-channel state, failure counts, backoff

### src/security_review.zig
- `SecurityCheckResult` struct: name, category, severity (doctor.Severity), message, score_impact
- `SecurityCheck` vtable struct: extensibility point with `run` fn pointer taking CheckContext
- 8 built-in check functions with consistent return pattern
- `calculateScore()`: starts at 100, deducts score_impact for non-ok results, clamped 0-100
- `calculateGrade()`: A (>=90), B (>=80), C (>=70), D (>=60), F (<60)
- `runAllChecks()`: executes all 8 checks, returns SecurityReport with score, grade, summary counts
- `formatReviewJson()`: JSON with score, grade, checks array, summary object
- `formatReviewText()`: human-readable with severity icons [ok]/[warn]/[ERR]

## Decisions Made

1. **Manual JSON writer**: Matched the existing `ReadinessResult.formatJson` pattern from health.zig using `std.ArrayList(u8)` + `writer.print`. No external JSON library needed.
2. **CheckContext over anytype**: The SecurityCheck vtable `run` function takes `*const CheckContext` instead of `anytype` because Zig fn pointers cannot use `anytype` parameters. This provides a clean extensibility contract.
3. **Direct calls in runAllChecks**: Built-in checks are called directly rather than through the vtable, keeping the hot path simple. The vtable struct exists for future user-defined checks.
4. **Score base 100 with deductions**: Only non-ok results deduct from score. OK results contribute nothing (they represent expected state). This means a fully-secure config gets 100, not some arbitrary sum.
