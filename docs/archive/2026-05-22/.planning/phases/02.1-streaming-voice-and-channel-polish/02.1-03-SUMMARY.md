## Implementation Details

### src/voice_mode.zig (new)

- **VoiceCapability** struct: `stt: bool, tts: bool` per-channel descriptor
- **channelSupportsAudio(channel)**: returns true if channel has any audio capability (replaces telegram hardcode)
- **resolveCapability(channel)**: case-insensitive lookup returning full capability struct
  - Audio-capable: telegram, discord, whatsapp, slack (full stt+tts)
  - No audio: cli, zaki_app, unknown channels
- **VoiceMode** struct: enabled flag, isActiveForChannel method, NarrationLabel constants (Listening.../Thinking.../Speaking...)

### src/observability.zig (modified)

- NarrationFrameType enum extended with `listening` and `speaking` variants
- All downstream consumers use `@tagName` which auto-handles new variants -- no switch exhaustiveness issues

### src/agent/root.zig (modified)

- Added `voice_mode` import
- `ttsAudioChannelSupported` body changed from `eqlIgnoreCase(channel, "telegram")` to `voice_mode.channelSupportsAudio(channel)`
- Function signature preserved -- all callers unchanged

### src/agent/narration.zig (modified)

- `turnStageToNarration`: voice_listening -> "Listening...", voice_speaking -> "Speaking..."
- `turnStageToFrameType`: voice_listening -> .listening, voice_speaking -> .speaking
