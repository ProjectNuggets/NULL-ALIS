## Honest gaps named (V1.10+ candidates)

These are real and observed during tonight's testing. None block the gate; all carry today's behavioral evidence.

### Gap A — `prose_survey` family filter widened ✅ CLOSED in `bdb187f`
**Was**: `fetchProseFactsByPattern` only matched `durable_fact/* | timeline_summary/* | summary_latest/*`. Panther zombies under user-keys invisible.
**Fix shipped**: Inverted to denylist — judge now sees ALL rows except audit (`autosave_*`, `session_checkpoint_*`), index (`timeline_index/*`), internal (`__tombstone__/*`, `last_hygiene_at`, `context_anchor_current`). Aligns with `classifyArtifactKey` taxonomy.
**Empirical proof**: SQL count comparison — same `WHERE content ILIKE '%codename%'` pattern returns **3 rows** under old allowlist, **12 rows** under new denylist. +9 user-keyed rows now visible to the judge.

### Gap B — `memory_archive` daily-type lockout ✅ CLOSED in `606ee16`
**Was**: `isEditableMemoryEntry` called `isMutableMemoryEntry` which gated on `category == .core`. Daily-type user-namespace keys were categorically refused by archive/forget/edit. Two Panther entries (`project_codename`, `project_codename_panther`) unkillable from agent side.
**Fix shipped**: Rewrote `isEditableMemoryEntry` to be category-agnostic — checks structural properties only (not internal, not system-managed, not append-only/tombstone/markdown-line). Plus added missing `timeline_summary/` to `isSystemManagedMemoryKey` (taxonomy comment said it was continuity but predicate didn't include it — pre-existing oversight).
**Empirical proof**: Live regression test — same `memory_archive key=project_codename` call that returned "protected" before now returns "Archived — soft-deleted, row preserved as audit, edges cascaded." Both previously-unkillable entries now archived.

### Gap C — Temporal-first judge reasoning ✅ CLOSED in `379cfc9`
**Was**: Judge applied "complementary, don't flag" rule too aggressively. Neptune→Nullalis codename rename went undetected because the judge saw "Architecture: Neptune (product) → nullALIS (agent core)" as legitimately complementary and didn't separately flag the older codename claim.
**Fix shipped**: System prompt restructured with numbered decision rules in priority order. Rule 1 (Temporal-first): when same property has different values across time AND the newer is ≥7 days newer (or explicitly a correction), the newer wins. Plus a worked example showing BOTH the correct flag AND the correct skip. Plus ISO date emission (YYYY-MM-DD) instead of unix epoch — Llama 8B reasons better about readable dates.
**Empirical proof**: New unit tests cover the date conversion (5 canonical timestamps including ZAKI's actual MNDA-delay April 16 and MNDA-signed May 6 rows). Live behavior: after the fix, ZAKI's narration of the codename history correctly identified Phase 1 (Neptune), Phase 2 (Nullalis rename), and Phase 3 (Panther = self-poll, archived) — all with dates and explicit correction-aware framing.
