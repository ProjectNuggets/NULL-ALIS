## 3. Honest analysis — why precision looks bad

The precision numbers are misleading at first read. Three sources:

### 3a. Judge over-strictness (artifact, not actual hallucination)
The precision judge compares each output statement against the ground-truth `facts[]` array. My ground-truth was written tight — only headline facts. When compaction faithfully extracts a supporting detail that's true in the conversation but absent from my narrow facts list, the judge marks it as hallucination.

Concrete example — `long_multi_topic/conv_01`:
- Output: *"Migration script is complete with `valid_to` bigint column and partial index"*
- Conversation: assistant said exactly this in turn 1
- Judge: "hallucinated" (because my `facts[]` only had high-level "migration_script.STATUS=complete")

This is a **ground-truth narrowness bug, not a compaction failure.**

### 3b. Genuine over-extraction on sparse conversations
`casual/conv_02` (pure pleasantries) — compaction emits 13 bullet points like *"User greeted the assistant"* and *"User said ttyl"*. These aren't hallucinations strictly, but they're meta-narrative noise. The user shouldn't see "User greeted assistant" appearing in their brain page.

**Real fix needed:** prompt should output `NO FACTS` (or empty) when no semantic content exists.

### 3c. Faithful but unsourced suggestions
`long_multi_topic/conv_01` — assistant offered alternative pricing ($49/$99/$200). User didn't accept. Compaction includes the suggestion as a fact. Judge flags it.

**Borderline case** — the suggestion was *discussed*, so it's recoverable from the conversation. But it's not a *user fact*. The right behavior is to mark it as `attributed_to: assistant` or omit. V1.6's `attributed_to` schema column addresses this; today's prompt has no signal.

## 4. The contradiction test — calibration, not failure

`error_recovery/conv_01` (Helix supersedes NeoVim, Asahi supersedes macOS):

Compaction output:
```
* User's favorite editor is Helix, switched from NeoVim last month.
* User's OS is Asahi Linux on M1 hardware, switched from macOS in March.
* Old memories of NeoVim and macOS are marked superseded.
```

Compaction **captured** the supersession. V1.6's contradiction LLM judge will then **resolve** it — close out NeoVim/macOS rows via bi-temporal `valid_to`. This is the canonical V1.6 capability test, and it's calibrated correctly: compaction does its job (extract the latest assertion); V1.6 does its job (resolve contradictions).

**No fix needed for this case at the compaction layer.**

## 7. Path decision: Path A — strengthen prompts

**Why not Path B (per-turn extraction):**
- Compaction recall is already 93% — the foundation works
- Per-turn would add inference cost on every turn for marginal recall gain
- The structural failures (precision, sparse-conversation hallucination) are addressable in the prompt

**Path A action plan:**

### A1. Compaction Pass C prompt strengthening
Two additions to the system prompt:
```
1. If the conversation contains no factual content (e.g., pure greetings,
   pleasantries, ack-only exchanges), respond with exactly: NO FACTS
2. Each bullet must be directly supported by the user/assistant text in
   the conversation. Do not include suggestions you offered, alternatives
   discussed but not adopted, or general commentary. Stick to what the
   user established.
```

### A2. Summarizer prompt degradation
Current summarizer forces all sections (focus/decisions/open_loops/...) even when content is sparse. Modify to:
- Make every section optional with `omit if empty` already in prompt — but the LLM ignores it. **Make the instruction louder:** "Output ONLY sections with substantive content. Empty sections must be omitted entirely, not 'None'."
- Add: "If the conversation has no fact content, respond with exactly: focus: chitchat\\n(end)"

### A3. Ground-truth corpus tightening
My `facts[]` arrays were too narrow. Add the supporting details to each ground truth so the precision judge has fair coverage. This isn't gaming the metric — it's making the metric measure compaction quality, not ground-truth-author thoroughness.

### A4. Judge prompt clarification
The precision judge needs explicit guidance: a statement that's *true given the conversation* but *not in the facts list* is "accurate-supplementary," not "hallucinated." Add this category.

## 9. What this baseline tells us about V1.6

1. **The compaction-derived extraction strategy (D2) is viable.** Foundation recall is strong.
2. **Summarizer prompt needs work.** May or may not survive — if Path A iteration can't rescue it, V1.6 carries extraction in Compaction Pass C alone (summarizer continues to emit prose summaries for `summary_latest` continuity but doesn't carry atomic-fact extraction).
3. **V1.6's contradiction LLM judge is necessary, not optional.** error_recovery/conv_01 showed compaction emits BOTH old + new versions of corrected facts — without V1.6's resolution layer, the brain page would show parallel contradictory entries.
4. **The `attributed_to` column in V1.6 schema is necessary.** Without it, suggestions vs decisions blur.
