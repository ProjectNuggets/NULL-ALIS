# V1.7 Frontend Handoff — Obsidian-equivalent UX recipe for zaki-prod

**Audience:** zaki-prod team rendering the `/brain` page.

**Premise:** the nullalis backend is **on par with Obsidian + ahead on 6 dimensions** (typed edges, bi-temporal, importance, auto-extraction, materialized edges, drilldown w/ archived rows). Side-by-side screenshots show zaki currently renders ~30% of what's in the payload. This doc maps each Obsidian power-feature to the backend field that drives it + concrete implementation guidance.

Everything below ships in V1.7. No backend work required to consume.

---

## The bar (Obsidian's specific moves we should match)

Look at Obsidian's graph view. The smoothness comes from FIVE things:

1. **Force-directed physics** with live tunable forces (center, repel, link, distance)
2. **Hover preview** — hover a node, see its summary in a tooltip without click/round-trip
3. **Color groups** by tag/path/search query — multiple groups simultaneously
4. **Live filter panel** — toggle orphans, control link thickness, node size, text fade
5. **Smooth zoom + pan** on infinite canvas with momentum

zaki's current render is missing all five. Each is FE work that maps cleanly onto already-shipped backend payload fields.

## Endpoint → UX mapping

### `GET /api/v1/users/:id/brain/graph` — global graph view

**Backend payload fields per node:**
```json
{
  "id": "extracted_a3f9",
  "kind": "core" | "daily" | "conversation",
  "session_id": "agent:zaki-bot:user:7:thread:main" | null,
  "created_at": 1714521600,
  "summary": "first 200 chars of content, UTF-8-safe truncated",
  "valid_to": null | 1714600000,
  "importance": 0.847,
  "display_label": "user PREFERS Helix" | absent,
  "community_id": 123 | null,
  "community_name": "Engineering work" | null
}
```

**Backend payload fields per edge:** `{type, source, target [, weight, predicate]}` where `type ∈ {session, semantic, reference, typed}`.

**FE moves to make:**

| Obsidian move | Backend field that drives it | Implementation |
|---|---|---|
| Color by group | `community_id` + `community_name` | Color palette indexed by `community_id mod 12`. Show `community_name` in legend. |
| Hover preview | `summary` + `display_label` + `importance` | On `mouseenter`, render tooltip from the EXISTING node object — NO round trip. `display_label` if present, fallback to `summary`. |
| Node size | `importance` (0..1) | Radius = `8 + 16 * importance`. Three-decimal float gives 1000 buckets — plenty. |
| Color by node kind (current) | `kind` | Keep as fallback when no community_id. |
| Color by edge type | `edges[].type` | Solid for typed, dashed for semantic, dotted for reference, faint for session. |
| Edge label | `edges[].predicate` (typed only) | Show on hover or zoom-in. |
| Show "archived" badge | `valid_to` not null | Dim node + badge icon. |
| Force-directed layout | n/a — pure FE | **Cytoscape.js** (recommended for graph features) or **D3 force**. Cytoscape.js param starting points: `gravity=0.4`, `nodeRepulsion=8000`, `idealEdgeLength=120`. |

**Query params zaki should send by default:**
```
GET /brain/graph?max_nodes=300&exclude_orphans=true
```
- `max_nodes=300` keeps render fast (zaki's screenshot showed it returning hundreds of orphan dots — overwhelming)
- `exclude_orphans=true` (V1.7a-8b) DEFAULTS to off but FE should default ON to match Obsidian's default

**Query params zaki should expose as live filters:**
- `search=foo` — intersects with `recallMemories` (BM25 + key + content). Reuse the same DSL as `/brain/search`.
- `link_types=preference,attribute` — CSV of LinkType values. Multi-select pill UI.
- `exclude_orphans=true|false` — toggle.
- `since=<unix>` — date floor (existing).
- `node_kinds=core,daily` — kind filter (existing).

### `GET /api/v1/users/:id/brain/local-graph?center_key=X&depth=N&max_nodes=M`

The depth-slider drilldown. Click any node in the global view → switch to local graph centered on that node.

**Default depth:** 1 (immediate neighbors). Slider 1-3.
**Default max_nodes:** 50.

**Backend returns:** `{center_key, depth, nodes:[{key, kind, hop_distance, score, summary, valid_to, session_id, link_type}], edges:[{source, target, predicate, weight, link_type}], stats:{nodes, edges}}`

**FE moves:**
- Show center node larger + highlighted
- Color other nodes by `hop_distance` (closer = more saturated)
- Edge styling identical to global graph
- "Back to global" button

---

### `GET /api/v1/users/:id/brain/orphans?limit=N`

The "lost notes" rail to the right of the graph (Obsidian default UI element).

**Backend returns:** `{orphans:[{id, key, kind, created_at, session_id, summary, valid_to, link_type}], stats:{orphans, limit}}`

**FE moves:**
- Vertical scrollable list
- Each row shows `summary` (truncate to 1 line) + `kind` badge + age (`created_at`)
- Click any orphan → opens drilldown OR centers local-graph on that node (which will likely be just the center, since it's an orphan)
- Default `limit=50`; "load more" button

---

### `GET /api/v1/users/:id/brain/diff?date=YYYY-MM-DD`

The timelapse scrubber.

**Backend returns:** `{window:{from, to, date, window_days}, births:[{key, kind, summary, ...}], deaths:[{key, valid_to, summary, ...}], stats:{births, deaths}}`

**FE moves:**
- Date picker (default: today)
- Two columns: "Born this day" + "Archived this day"
- Each row clickable → drilldown
- "Animate" button: cycle through last 30 days at 1 day/sec, highlighting affected nodes in the global graph view
- This IS Obsidian's "timelapse" button equivalent

---

### `GET /api/v1/users/:id/brain/communities` + `POST .../recompute`

Cluster legend + manual recompute trigger.

**Backend returns (GET):** `{communities:[{community_id, member_count, generated_at, name, name_source}], stats:{communities}}`

**Backend returns (POST recompute):** `{stats:{edges_loaded, nodes_in_lpa, communities_found, members_assigned, llm_calls_succeeded, llm_calls_failed, fallback_names_written}}`

**FE moves:**
- Legend panel: list communities sorted by member_count DESC. Each row = colored swatch (matching graph node color) + name + count.
- Click a community → filter graph to just members of that community.
- Recompute button (small, in legend header) — calls POST. Show toast "5 clusters detected, 12 LLM-named." Re-fetches GET /brain/communities + GET /brain/graph.
- Handle 409 Conflict: another recompute in progress; show "Recompute already running" instead of erroring.

---

### `GET /api/v1/users/:id/brain/memory/:key` — drilldown

Single-fact deep view (existing).

**Already returns:** full row + edges + chronological event history + `archived: bool` (V1.7a-5b) + `link_type` (V1.7a-5).

**FE moves:**
- Show "archived" badge dimmed when `archived=true`
- Show `link_type` as a colored pill at top of the page
- Show edge list grouped by predicate (using existing `link_type` mapping)
- Add "Show local graph" button → switches to /brain/local-graph centered here

## The five Obsidian moves, concretely

### 1. Force-directed physics

Use **Cytoscape.js** with the `cose-bilkent` layout. Recommended params:

```js
const layout = {
  name: 'cose-bilkent',
  animate: true,
  animationEasing: 'ease-out',
  animationDuration: 600,
  randomize: true,
  nodeRepulsion: 8000,
  idealEdgeLength: 120,
  edgeElasticity: 0.45,
  gravity: 0.4,
  numIter: 2500,
};
```

Expose live sliders for: `nodeRepulsion`, `idealEdgeLength`, `gravity` (matches Obsidian's "repel force / link force / center force" panel).

### 2. Hover preview

Backend already returns `summary` (200 chars), `display_label`, `importance`, `community_name` per node. ~30 LOC of React:

```jsx
<Tooltip
  content={
    <div>
      <h4>{node.display_label || node.key}</h4>
      <p className="muted">{node.kind} · {node.community_name || 'no cluster'}</p>
      <p>{node.summary}</p>
      <small>importance {node.importance.toFixed(2)}</small>
    </div>
  }
  open={hoveredNode === node.id}
/>
```

NO ROUND TRIP. The data is already in the node object from the initial `/brain/graph` call.

### 3. Color groups

Two presets + user-toggle:
- **By community** (default): color = palette[community_id mod 12]
- **By link_type**: color = `linkTypeColors[node.link_type]` (use the 7 LinkType values)
- **By kind** (current zaki behavior): color = kindColors[node.kind] — keep as fallback

User can also add **search-based color groups** (Obsidian power-feature):
- Input: search query (uses backend `?search=`)
- Output: matching nodes get an override color

### 4. Live filter panel

Match Obsidian's settings panel layout (right sidebar):

```
Filters
  Search: [_______]
  Tags toggle: ON/OFF
  Existing files only: ON/OFF
  Orphans: ON/OFF (default OFF — backend filter when off)

Groups
  + New group
  [color] tag:#X     [pencil][trash]
  [color] path:Y     [pencil][trash]

Display
  Arrows: ON/OFF
  Text fade threshold: [slider]
  Node size: [slider]
  Link thickness: [slider]
  [Animate] button (timelapse)

Forces
  Center force: [slider]   → cytoscape gravity
  Repel force: [slider]    → cytoscape nodeRepulsion
  Link force: [slider]     → cytoscape edgeElasticity
  Link distance: [slider]  → cytoscape idealEdgeLength
```

Sliders mutate cytoscape config + `cy.layout(newConfig).run()`.

### 5. Smooth zoom + pan

Cytoscape.js gives this for free:
```js
const cy = cytoscape({
  container: ref.current,
  wheelSensitivity: 0.2,    // smooth zoom
  minZoom: 0.1,
  maxZoom: 4.0,
  // pan is enabled by default
});
```

Don't fight it. The current zaki render likely uses a non-physics layout (grid? circular?) which is why it feels static.

## Performance guidance

zaki's current screenshot shows hundreds of orphan dots. The backend can help:

| Symptom | Backend fix | FE fix |
|---|---|---|
| Hundreds of orphan dots dominate | Pass `?exclude_orphans=true` (V1.7a-8b) | Default toggle ON |
| Slow initial render | Pass `?max_nodes=300` | Add "show more" button to bump to 500 |
| Graph re-renders on every scroll | n/a | Memoize cytoscape instance, only re-render on filter change |
| Hover lags | n/a — payload already complete | Use the existing node data; no extra fetch |
| Canvas pan stutter | n/a | WebGL renderer in cytoscape (`cy.renderer('canvas-webgl')`) for >500 nodes |

## Acceptance: what "Obsidian-quality" means for zaki

Before we call zaki's `/brain` page shipped:

- [ ] Force-directed layout (cytoscape or d3) with live tunable forces
- [ ] Hover preview from cached node payload (no round trip)
- [ ] Color presets: community / link_type / kind (user toggle)
- [ ] Search box that calls `/brain/graph?search=...`
- [ ] Filter pills for `link_types` (7 LinkType values)
- [ ] Orphan toggle (default ON; calls `?exclude_orphans=true`)
- [ ] Orphan rail consuming `/brain/orphans`
- [ ] Local-graph drilldown (click node → fetch `/brain/local-graph?center_key=X`)
- [ ] Cluster legend consuming `/brain/communities` with recompute button
- [ ] Time scrubber consuming `/brain/diff?date=` with animate
- [ ] Smooth zoom + pan (cytoscape default; no overrides)
- [ ] Default `max_nodes=300` on initial load

When all 12 are checked, zaki's `/brain` will feel as smooth as Obsidian, with three superpowers Obsidian doesn't have:
1. **Auto-extracted facts** (no manual link curation needed)
2. **Bi-temporal close-out** (history is preserved, archived rows browsable)
3. **LLM-named clusters** (Obsidian requires plugins; we ship native)

## Known limitations to communicate to users

- **Communities require recompute.** Until the user POSTs `/brain/communities/recompute` (or a nightly job runs), `community_id` will be null on all nodes. Show a "Compute clusters" CTA when `/brain/communities` returns empty.
- **LLM-naming requires sidecar provider.** When sidecar isn't configured, recompute writes fallback names ("Cluster 47312"). The `name_source` field tells you which: `"llm"` vs `"fallback"`. FE can show fallback names in italic or dim them.
- **One recompute at a time.** POST returns 409 Conflict on overlap. UI should disable the recompute button while one is in flight.
