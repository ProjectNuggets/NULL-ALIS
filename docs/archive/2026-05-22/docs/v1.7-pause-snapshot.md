---

## Why we paused

V1.7-ship was 4-of-4 internal phases away from `git push + tag v1.7.0`. Pre-tag
diagnostic discovered **0 typed edges across the entire database** (16K+ memories
across 13 user_ids, all with `active_edges = 0`). The graph view we built (V1.7a-9
communities, V1.7a-7 local-graph, V1.7a-8a orphans, V1.7a-8b filters) is
**structurally complete but user-empty** — backend correctly renders the
empty-database truth, but the value proposition assumes edges exist.

Tagging "Obsidian-parity per user" while every user sees a starfield = false
advertising. So we stop, build the memory pyramid that produces edges, then
resume V1.7 ship.

The 0-edges discovery is itself a finding worth keeping — it proves the
behavioral-audit method (you find what code-review can't) and motivates the
V1.8 audit-first methodology.

## V1.8 reframe — why this pause is the right move

**Next-generation principle (Nova 2026-05-04, locked as auto-memory)**: every commit
asks "is this what the next generation would build, knowing what we know in 2026,
or just what fits the constraints we inherited?" V1.7 was almost what 2024 would
build. V1.8 is what 2026 would build.

**The pyramid we're building toward** (preview from Phase 3):

```
L8 Markdown mirror     — human-readable surface
L7 Identity            — durable core (preferences, persona)
L6 Procedural          — learned patterns + corrections
L5 Episodic            — session/daily/weekly rollups
L4 Graph               — typed edges + entities + communities
L3 Vectors             — semantic similarity
L2 Atomic facts        — extracted (memories table)
L1 Working memory      — current turn (in-RAM)
L0 Raw stream          — every message + tool call (append-only)
```

V1.7 covered L1-L4 and L8 (partially). V1.8 will:
- Make L2 LIFT reliable (cheap deterministic pass + LLM pass + nightly rollup)
- Make L0→L2 lift policy explicit (so messages aren't write-only)
- Add L5 daily/weekly rollup layer (currently missing)
- Add L6 procedural-memory layer (we have nothing here)
- Solidify L3 coverage (every L2 row vectorizes; verify in audit)
- Solidify L4 production reliability (the 0-edges fix)
