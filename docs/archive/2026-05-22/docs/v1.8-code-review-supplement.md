## V1.8-11 — LPA two-table fix (CRITICAL FINDING)

**Files:** `src/zaki_state.zig` (+27 / -8)

This was the biggest finding from the deep cleanup audit. A SHIPPED warm-context block (`<active_communities>`) was returning null on every turn because of a real architectural mismatch.

### Root cause
- Extraction writes edges with `target_key` = entity_id (32-char hex from `upsertEntity` → `memory_entities` table)
- `listMemoryEdgesForCommunityCompute` required BOTH endpoints to exist as rows in `memories`
- All 33 distinct target_keys were missing from `memories` (they live in `memory_entities`)
- SQL returned empty → LPA had no input → 0 communities computed → `<active_communities>` always null

### Findings

| # | Severity | Finding | Disposition |
|---|---|---|---|
| 9 | 🔴→✅ | LPA two-table mismatch. Pre-V1.8-11: 0 valid LPA edges. Post-fix: 32 valid LPA edges, 24 LLM-named communities, 32 memories tagged with `community_id`, agent demonstrably reads them. | Closed by V1.8-11 |
| 10 | 🔵 | Fix uses `OR EXISTS (... memory_entities ...)` on both endpoints. Adds 1 EXISTS check per endpoint. Indexed on `(user_id, id)` in `memory_entities`. Marginal cost. ✓ | Working as designed |
| 11 | 🔵 | `setMemoryCommunityIds` left untouched: still only writes `community_id` to `memories`. Entity endpoints participate in label propagation as connective tissue but don't receive community labels themselves. Correct: communities are user-facing memory-row-scoped concepts; entities are structural. | Working as designed |
| 12 | 🔵 | Comment block in the SQL function documents the V1.8-11 rationale. Future-me reading this in 6 months will know why the OR-EXISTS shape is necessary. ✓ | Working as designed |

### Smoke evidence
- `/api/v1/users/7777/brain/communities/recompute`: `{"edges_loaded":32,"nodes_in_lpa":56,"communities_found":24,"members_assigned":56,"llm_calls_succeeded":24,"llm_calls_failed":0,"fallback_names_written":0}`
- `memory_communities` row count: 0 → 24
- `memories` rows with `community_id != NULL`: 0 → 32
- Top communities by member_count: "Helios launch deadline" (4), "Velox Industries meetings" (3), "Database refactor timeline" (3), etc.
- Agent in smoke test reasoned about all 24 communities by name when asked
