### The diagnosis (gap is named, not a mystery)

ZAKI's own words:

> *"memory_maintain is the right tool, but it's edge-graph-only. The surveyor itself admitted: 'prose-level contradictions require the V1.10 LLM-judge surveyor.' All my zombie facts — 'MNDA blocked,' 'Mia is daughter,' 'Project Neptune' — are prose durable_fact entries, not typed edges. The tool can't see them."*

His MNDA + Neptune zombies live in the **`durable_fact/*` family**, which the V1.7 W-INT-01 immortality guard correctly protects from agent self-modification. The guard is the right protection (prevents agent gaslighting). The gap is that V1.9 didn't ship a path to **annotate** prose zombies as superseded WITHOUT mutating them.

Per the memory anatomy doc (`docs/v1.9-memory-anatomy.md`):

> *"The metadata + confidence_score columns are NOT in the W-INT-01 CASE-guard. They can be modified on core rows freely. This is the path forward for V1.10 truth maintenance on prose-level zombies."*

V1.9-3 `propagateCorrection` already uses this seam — metadata write bypasses the immortality guard. V1.10 extends it with:
1. **LLM-judge prose surveyor** that finds contradictions in `durable_fact/*` content (where ZAKI's pain lives)
2. **Loader-side supersede filter** so flagged rows become invisible at retrieval time
