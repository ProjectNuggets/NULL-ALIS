# S14.10 — License audit (2026-04-26)

**Scope:** all `build.zig.zon` deps + Dockerfile base images. zaki-infra k8s images audited separately on the zaki-infra side.
**Audited at commit:** `224bea5`
**Goal:** no GPL/AGPL surprises, no untracked transitive license obligations.

## nullalis own license

`LICENSE` file at repo root: **nullALIS Dual License (Provisional)** — internal own license, not OSI-listed. Out of scope for this audit (it's the licensor side).

## Zig dependencies (`build.zig.zon`)

| Dep | URL | License | Risk |
|---|---|---|---|
| `sqlite3` | `git+https://github.com/allyourcodebase/sqlite3#8f840560eae88ab66668c6827c64ffbd0d74ef37` | **Public Domain** (sqlite itself); the wrapper at `allyourcodebase/sqlite3` is **MIT** | ✅ Low — sqlite is public domain (no copyleft); MIT wrapper imposes minimal attribution |
| `sentry_zig` | `https://github.com/nullclaw/sentry-zig/archive/refs/tags/v0.1.0.tar.gz` | Hosted under `nullclaw` (own org); license should be MIT or own | ✅ Low — verify upstream LICENSE file is permissive |

Both deps are **SHA-pinned via `.hash`** (build.zig.zon native pinning); supply chain hash mismatch would fail the build. This already addresses S16.6 dep SHA pinning for these two deps.

## Dockerfile base images

| Image | Stage | License | Risk |
|---|---|---|---|
| `alpine:3.23` | `builder`, `release-base`, `release` | **MIT** (musl libc) + various component licenses (busybox under GPL-2.0 — but unused at runtime in nullalis since the Zig binary is musl-static-linked) | ✅ Low — Alpine itself is permissive; busybox GPL is per-binary, not per-image |
| `busybox:1.37` | `config` (build-time only, not runtime) | **GPL-2.0** | ⚠️ Build-time only — used to copy config.example.json into the image; the resulting nullalis runtime image does NOT bundle busybox. Per GPL-2.0, build-time use of a GPL tool to produce a non-derivative artifact is permitted |

## Runtime binary deps (statically linked into nullalis)

| Lib | Source | License |
|---|---|---|
| musl libc | bundled by Zig's `-target *-linux-musl` builds | **MIT** |
| Postgres libpq | linked from Alpine `postgresql-dev` package | **PostgreSQL License** (BSD-style) |

## Findings

✅ **No GPL/AGPL/LGPL contamination in the runtime artifact.** The only GPL-2.0 component (busybox in the `config` build stage) is build-time-only and does not produce derivative code in the runtime image.

⚠️ **`sentry_zig` license — VERIFIED MISSING (S14.10.1, 2026-04-26).** The hash-pinned tarball at `github.com/nullclaw/sentry-zig` has **NO LICENSE file**. Verified via:
- `curl -sIL https://raw.githubusercontent.com/nullclaw/sentry-zig/main/LICENSE` → HTTP 404
- `curl -s https://api.github.com/repos/nullclaw/sentry-zig` → `"license": null`
- Root directory listing: `.gitignore`, `README.md`, `build.zig`, `build.zig.zon`, `docs/`, `src/`, `tests/` — no LICENSE / COPYING / LICENSE.md

**Implication:** under default copyright law, code with no declared license = "all rights reserved." We technically lack permission to redistribute (build-time inclusion in nullalis binary is a redistribution). Same-org operator (nullclaw is Nova-controlled) so risk is low, but hygiene gap is real.

**Operator action required:** Nova adds an MIT or Apache-2.0 LICENSE file at `nullclaw/sentry-zig`. ~5 min UI action via GitHub web. Once landed, update this audit with the LICENSE type + commit SHA.

✅ **All deps are hash-pinned** — no moving-ref supply chain risk for the two declared deps. This satisfies the S16.6 dep SHA pinning DoD for these specific entries; transitive deps (any sub-deps of sqlite3-zig or sentry-zig) inherit the pin via Zig's `--system` integrity check.
