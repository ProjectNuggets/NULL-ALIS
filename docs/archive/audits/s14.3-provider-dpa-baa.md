## Findings

**HIGH-equivalent:** Together AI carries the active LLM workload (every chat turn → Together). DPA absence is the most significant gap. Action: operator submits DPA request via Together's portal.

**MED:** Composio runs per-tenant tool execution. Mixed user data passes through (search queries, document content, etc). DPA required if any EU/CCPA-scope user is in the user base.

**LOW:** Sentry DSN is shipping error contexts that may contain limited user identifiers (session_id, user_id). DPA needed for full compliance but operational risk is low (errors are short-lived in Sentry retention).

## What's NOT in this audit

- **HIPAA / BAA:** none of the listed providers offer BAAs without enterprise contracts. nullalis is not advertised as HIPAA-scope today; if it ever is, all providers above need re-evaluation. Track as `S14.3.1` (trigger: HIPAA scope decision).
- **EU AI Act:** S14.2 covers EU AI Act risk classification separately.
- **Provider-side data deletion:** D25/D33 cover GDPR purgeUser on the nullalis side. Provider-side deletion (does Together/Composio/Sentry honor a delete request for user-derived data?) is a separate concern, tracked here as `S14.3.2` (trigger: first GDPR delete request from a paying user).
