# S0.5 Backfill Report (2026-03-12)

## Objective
Introduce a deterministic, high-confidence Telegram identity backfill path before strict canonical enforcement expands.

## Scope Implemented
- Added backfill runner: `src/channel_identity_backfill.zig`
- Added deterministic identity-key builder: `src/channel_identity_key.zig`
- Added cache invalidation hooks in canonicalizer:
  - `invalidateAllCache()`
  - `invalidateCacheForIdentity(...)`

## Backfill Logic (High Confidence Only)
For each Telegram candidate from Postgres `channel_state`:
- `missing`:
  - empty/invalid `account_id`
- `ambiguous`:
  - `chat_id <= 0` (group/scope ambiguity)
  - conflicting candidate users for same `account_id + chat_id`
  - existing binding resolves to a different user
- `migrated`:
  - private-chat candidate (`chat_id > 0`) with non-conflicting mapping

No guess-binding is performed.

## Manual Review Queue Rules
Review all rows classified as:
- `non_private_chat_scope`
- `conflicting_candidate_users`
- `binding_conflict_existing_user`
