# nullalis Status Snapshot — 2026-03-06

This document is the operational truth source for the current `heartbeat-config-source` branch before merge to `main`.

## Scope in this branch

- Heartbeat runner + wake hook and config-source fixes.
- Scheduler/proactive guardrails and diagnostics improvements.
- Telegram webhook/reply reliability stabilization.
- ZAKI SSE contract alignment for chat stream.
- Postgres preflight/build hardening.
- Web search provider override wiring (Brave/Exa).
- Composio v3 endpoint/payload compatibility fixes.

## Most Important Reliable Truths

1. Postgres state and markdown memory are intentionally separate.
- `state.backend=postgres` controls scheduler/session/channel/secrets state plane.
- Memory can still resolve to markdown by design.
- Agent saying "memory backend is markdown" does not mean Postgres state is off.

2. Startup self-check is the source of truth for runtime state.
- Use gateway log line `startup.self_check`.
- Healthy target fields:
  - `state_effective=postgres`
  - `scheduler_backend=postgres`
  - `degraded=false`

3. Telegram reliability depends on per-user token resolution and valid token shape.
- Nullalis now prefers tenant-scoped token file with DB sync fallback.
- Invalid token shape (missing `:` separator) is rejected early.
- Inbound and outbound are expected on the same user-scoped main session.

4. Heartbeat currently operates with global enable + per-user config.
- Global heartbeat disable blocks polling even if user heartbeat files are enabled.
- Legacy per-user interval seconds keys are accepted.
- Cadence currently resolves in minutes at runtime.

5. Composio integration now matches v3 API contract shape.
- List filter uses `toolkit_slug`.
- Execute path uses `/api/v3/tools/execute/{tool_slug}`.
- Connect payload includes both `user_id` and `auth_config_id`.
- Tool slug normalization preserves returned slug shape (no forced kebab-case rewrite).

## Current Open Ends

1. Heartbeat architecture still needs scale-oriented design finalization.
- Current behavior is stable for dogfood.
- Long-term plan should define queue/lease behavior for higher tenant counts.

2. Composio credential model is still global-by-config.
- Works now.
- Recommended target remains per-user secret wiring for multi-tenant isolation.

3. UI rendering still does not expose all intermediate/tool timeline states.
- Terminal logs can show richer intermediate reasoning/tool context than current chat UI rendering.

4. Legacy noisy test users/jobs can pollute local logs.
- Archive/cleanup test users/jobs in local dev to keep diagnostics readable.

5. Turn origins `wake` and `proactive` are reserved for v0.2 wiring.
- Current runtime assigns `user`, `heartbeat`, and `scheduler`.
- v0.2 origin wiring plan is documented in `docs/v0.2-origin-roadmap.md`.

## Tips and Tricks

1. If app works but Telegram does not:
- Check token-shape warnings first.
- Check webhook URL/user_id routing.
- Verify only one active nullalis gateway instance for the target endpoint.

2. If agent appears “confused” about infra capabilities:
- Confirm runtime facts from logs, not assistant prose.
- Startup self-check line is authoritative.

3. If a turn stream shows ownership lock conflict:
- It can clear after lease timeout.
- Treat it as control-plane contention, not tool failure.

4. If logs look active “without prompts”:
- Background heartbeat/scheduler/wake processing can legitimately produce turns.
- Verify source via diagnostics endpoint and event logs.
