# V0.7 BFF Lock Retry Contract

## Purpose

Define how BFF handles gateway tenant ownership lock conflicts so end users do not see raw lock errors.

Scope: applies to lock-protected write/chat routes that can return `ownership_lock_conflict`.

## Gateway Conflict Contract (Input to BFF)

HTTP status: `409 Conflict`  
Header: `Retry-After: <seconds>`  
Body:

```json
{
  "error": "ownership_lock_conflict",
  "message": "user is active on another node, retry shortly",
  "retry_after_ms": 250,
  "owner_instance_id": "node-a",
  "lease_until_s": 1760000000
}
```

SSE conflict events include:
- `code=ownership_lock_conflict`
- `retry_after_ms`
- optional `owner_instance_id`
- optional `lease_until_s`

## BFF Retry Policy

1. Retry only when:
- status is `409`
- `error == ownership_lock_conflict`
2. Retry budget:
- max wall time: `1500ms`
- max attempts: `3`
3. Backoff:
- use gateway `retry_after_ms` when present (clamp `50..1000ms`)
- fallback sequence: `100ms`, `250ms`, `500ms`
- add jitter ±20%
4. Idempotency:
- preserve inbound idempotency key across retries
- do not mutate request payload between retries

## Failure Mapping (BFF -> Client)

After retry budget exhausted:

```json
{
  "error": "temporary_contention",
  "message": "Agent is busy on another node. Retry shortly."
}
```

Status: `503 Service Unavailable`

## Logging and Metrics

For each retried request, BFF emits:
- `lock_conflict_detected` (counter)
- `lock_conflict_retries_attempted` (counter)
- `lock_conflict_retry_success` (counter)
- `lock_conflict_retry_exhausted` (counter)
- latency histogram from first attempt to final outcome

Log fields:
- `user_id`
- `route`
- `attempt`
- `retry_after_ms`
- `owner_instance_id` (when present)
- `final_status`

## Safety Constraints

1. BFF must never drop `X-Zaki-User-Id` between retries.
2. BFF must preserve auth and internal token boundaries.
3. BFF must not retry non-lock errors (`4xx` other than lock conflict, or `5xx`).

## Rollout

1. Shadow mode:
- metrics/logging enabled, retries disabled
2. Enable retries for internal cohort
3. Enable retries for all cohorts after SLO check
4. Rollback by flipping retry feature flag off
