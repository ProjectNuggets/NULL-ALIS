# nullALIS v0.1 Public Posture

Date: 2026-03-25
Status: current public posture freeze

This document freezes the validated runtime and deployment posture that should
be treated as current truth for `nullALIS v0.1`.

Use this document for:
- operator onboarding
- local/dev runtime setup
- product/runtime truth checks
- deciding whether older reports are still current

Do not use this document as a rollout log. Historical rollout evidence remains
under `docs/reports/`.

## Validated Runtime Baseline

- Profile: `zaki_bot`
- Primary chat model: `together-ai/moonshotai/kimi-k2.5`
- Chat fallback chain: `openrouter`
- Embedding provider: `together-ai`
- State backend: `postgres`
- Scheduler backend: `postgres`
- Runtime posture: `degraded=false`

Validated local benchmark posture on this branch:
- session lifecycle crash fix applied
- local benchmark completed without crash
- local parallel tool dispatcher validated with `parallel_tools=true`,
  `tool_dispatcher=parallel`, `parallel_tools_rollout_percent=100`

What this proves:
- current local runtime boots cleanly on latest validated code
- the observed session eviction crash path is fixed
- the parallel tool path can complete a real local benchmark run without
  reproducing the crash

What this does not prove:
- long-duration soak stability
- production cluster behavior under sustained concurrency
- absence of unrelated session/contention bugs

## Deployment Posture

Current product posture is internal-service first:
- `nullALIS` runs as the internal agent service behind `zaki-prod`
- `zaki-prod` owns browser/backend proxy behavior
- `zaki-infra` owns production deployment, Helm values, ArgoCD rollout, and
  secret/config wiring

Operational rule:
- treat `deploy/k8s/zaki-bot/` as reference-only
- do not treat that pack as the live production source of truth

Telegram/public webhook posture:
- direct public Telegram webhook delivery to Nullalis is not the current
  baseline truth
- app-edge/internal relay work remains a separate follow-up track

## Local And Dev Requirements

Required local/runtime contract for `zaki_bot`:
- provider entries under `models.providers` must include `base_url`
- `agents.defaults.model.primary` must resolve to a configured provider entry
- internal service token must be non-placeholder
- placeholder values such as `dev-internal-token`, `test-internal-token`,
  `changeme`, `default`, and `REPLACE_WITH_*` are invalid

Typical local ports:
- `nullalis` gateway: `127.0.0.1:3000`
- `zaki-prod` backend: `127.0.0.1:8787`
- `zaki-prod` app frontend: Vite dev port (commonly `5173`)

Recommended local verification order:
1. `zig build test --summary all`
2. start gateway with `scripts/gateway-clean.sh --profile ops`
3. verify `/health`
4. verify `/internal/diagnostics`
5. verify app-to-agent authenticated flow through `zaki-prod`
