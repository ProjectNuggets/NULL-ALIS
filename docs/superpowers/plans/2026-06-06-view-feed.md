# In-App View-Feed (gateway contract for Zaki) — Implementation Plan (Plan 5 of 8)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Let the user *watch* the agent browse. The orchestrator exposes a per-session **frame** (screenshot + url + title); after each browser action the gateway tools emit a **`browser_frame`** event on the existing per-turn SSE/observer bus, so the Zaki app renders the agent's browsing live — reusing the user-scoped stream Zaki already consumes (no new endpoint, no new authz).

**Architecture:** Three pieces. (1) Orchestrator `GET /v1/sessions/{id}/frame → {frame: base64-png, url, title}` (lazy capture: `screenshot` + `get url` + `get title`). (2) The Zig `browser_*` tools, after a successful navigate/snapshot/exec, fetch a frame via the `OrchestratorClient` and emit a `browser_frame` observer event (the tools already receive the gateway `event_bus` in `allTools` opts). (3) A documented contract for Zaki. **Watch-only** (no input forwarded — co-browse is a non-goal). The Zaki UI itself is built in the Zaki repo against the contract here.

**Tech Stack:** Go (orchestrator frame endpoint), Zig (tools + the `browser_frame` event on the existing observability/SSE bus), k3d for the e2e.

> **References:** spec §9a/§11a (view-feed contract, watch-only, `browser_frame` event reusing `StreamCallback`/`ObserverEvent`); the per-user authz is inherited from the chat SSE stream (no separate `owner==subscriber` check needed — the stream is already scoped to the conversation/user).

- [ ] **Step 2: Implement `Frame`** (`k8s_provider.go`): screenshot to a temp path, base64-read it (binary-safe, via the existing `execRaw`), and read url/title:
```go
func (p *K8sProvider) Frame(ctx context.Context, sessionID string) (Frame, error) {
	podName, ok := p.reg.Pod(sessionID)
	if !ok {
		return Frame{}, fmt.Errorf("unknown session %q", sessionID)
	}
	if _, err := p.Exec(ctx, sessionID, []string{"screenshot", "/tmp/vf.png"}); err != nil {
		return Frame{}, fmt.Errorf("screenshot: %w", err)
	}
	b64, err := p.execRaw(ctx, podName, []string{"sh", "-c", "base64 /tmp/vf.png 2>/dev/null"}, nil)
	if err != nil {
		return Frame{}, fmt.Errorf("read frame: %w", err)
	}
	urlRes, _ := p.Exec(ctx, sessionID, []string{"get", "url"})
	titleRes, _ := p.Exec(ctx, sessionID, []string{"get", "title"})
	return Frame{
		PNGBase64: strings.TrimSpace(b64),
		URL:       strings.TrimSpace(urlRes.Stdout),
		Title:     strings.TrimSpace(titleRes.Stdout),
	}, nil
}
```
(`get url`/`get title` are read verbs already on the allowlist; the URL guard only fires on navigation verbs, so this is unaffected. Add `stubProvider.Frame` returning a canned Frame.)

- [ ] **Step 3: HTTP endpoint** (`server.go`): register `s.mux.HandleFunc("GET /v1/sessions/{id}/frame", s.handleFrame)`:
```go
func (s *Server) handleFrame(w http.ResponseWriter, r *http.Request) {
	ctx, cancel := context.WithTimeout(r.Context(), 30*time.Second)
	defer cancel()
	f, err := s.provider.Frame(ctx, r.PathValue("id"))
	if err != nil {
		writeJSON(w, http.StatusInternalServerError, map[string]string{"error": err.Error()})
		return
	}
	writeJSON(w, http.StatusOK, f)
}
```

## Task 4: emit a frame after each browser action

**Files:** Modify `src/tools/browser_navigate.zig`, `browser_snapshot.zig`, `browser_exec.zig`, `src/tools/root.zig`.

- [ ] **Step 1:** Give the three action tools an `event_bus: ?*bus.Bus = null` field (the bus type used by `allTools` opts — `opts.event_bus`). In `allTools`, set `.event_bus = opts.event_bus` when constructing `browser_navigate`/`browser_snapshot`/`browser_exec` (read the existing `event_bus` plumbing — other tools already receive it; mirror them). `browser_new_session`/`close_session` don't emit frames.

- [ ] **Step 2:** After a SUCCESSFUL action (success path of `execute`, before returning), if `self.event_bus` is non-null, fetch a frame (`self.client.getFrame(allocator, sid)`), parse `{frame,url,title}`, and publish a `browser_frame` ObserverEvent on the bus (best-effort: on any frame error, log + skip — never fail the tool because the view-feed hiccuped). Factor this into a small shared helper (e.g. in `browser_exec.zig`, `pub fn emitFrame(allocator, bus, client, session_id) void`) and call it from all three tools. (Frame capture adds latency; that's acceptable for the watch UX. A future optimization: emit asynchronously.)
