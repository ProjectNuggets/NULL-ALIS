## 5) Rollout Vs Rollback Policy

Rollout (allow):
1. build/test gates pass
2. profile-B gates pass for active stage
3. error rate `< 1%`
4. no sustained lock-conflict/overload spikes

Rollback / hold (stop promotion):
1. profile-B breach in active stage
2. error rate `>= 1%`
3. sticky routing drift or ownership backend mismatch in staging
4. sustained queue saturation and rising p95 trend across consecutive windows

Scale-claim release gate (must pass before public claim):
1. profile-A latency
2. isolation gate
3. staged multi-cell reproducibility
