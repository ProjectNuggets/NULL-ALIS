# v0.7 Backlog (Execution Order)

## Goal
Ship a production-ready multi-user agent experience with reliable Telegram voice, controllable UX settings, safe isolation boundaries, and clear rollout gates.

## Priority Queue
1. `V0.7-T1` Telegram STT reliability (P0)
- Ensure voice/audio inbound never silently drops.
- Enforce deterministic fallback reply when transcription fails.
- Add diagnostics checks for `transcription_attempted/succeeded/failed/skipped_no_transcriber`.

2. `V0.7-T2` Vision enablement with capability gating (P0)
- Keep image ingestion active (`[IMAGE:...]` markers).
- Add provider/model allowlist for vision-capable models.
- Return deterministic user error when model is non-vision.

3. `V0.7-T3` Telegram formatting parity (P1)
- Add channel-specific renderer for Telegram-safe formatting.
- Normalize list/code/bold output and chunk long replies.

4. `V0.7-T4` File isolation hardening (P1)
- Keep shell/git sandbox fail-closed when sandbox is enabled.
- Verify workspace path restrictions for tool execution in multi-user deployment.
- Define staging acceptance test for cross-tenant read/write denial.

5. `V0.7-T5` User config mapping (P1)
- Expose product-level settings schema (not raw runtime knobs).
- Map settings deterministically to runtime config patch payloads.
- Keep slash commands as runtime-only overrides.

6. `V0.7-T6` Productization completion (P1)
- Finalize BFF onboarding flow: provision, connect Telegram, test ping, settings save.
- Add usage state and soft-limit warnings in user-facing API.

7. `V0.7-T7` Rate-limit and queue policy verification (P1)
- Validate provider and runtime rate limiting under concurrent load.
- Confirm queue/drop/debounce semantics match configured policy.

8. `V0.7-T8` Agent behavior tuning to full potential (P2)
- Run prompt/tool behavior sweep per channel.
- Tune defaults for response quality, stability, and cost.
