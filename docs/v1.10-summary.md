## Honest gaps named (V1.10+ candidates)

These are real and observed during tonight's testing. None block the gate; all carry today's behavioral evidence.

### Gap A — `prose_survey` family filter is too narrow
`fetchProseFactsByPattern` only matches `durable_fact/* | timeline_summary/* | summary_latest/*`. The Panther zombies live in user-namespace keys (`project_codename`, `project_codename_panther`, etc.) — invisible to the surveyor.

**Fix shape**: extend the family filter to include user-keyed memories, or expose a `families: string[]` parameter on `prose_survey`. ~30 LOC + careful default selection (don't accidentally judge `project_settings`, `user_preferences`, etc.).

### Gap B — `memory_archive` refused 2 daily-type entries
`project_codename_panther` and `project_codename` are `memory_type=daily, is_latest=true, valid_to=NULL`. memory_archive said "protected" on both. memory_forget refused too. This is NOT the W-INT-01 immortality guard (which is core-only per `docs/v1.9-memory-anatomy.md`).

**Fix shape**: investigate the protection mechanism. Most likely a stale guard or a missing branch in `memory_archive`'s SQL. ~1 hour of code-spelunking.

### Gap C — Judge needs temporal reasoning to disambiguate sequential truths
Tonight's `prose_survey entity_pattern=Neptune max_facts=50` returned `contradictions_found=0` AND `more_available=true`. The judge saw "Architecture: Neptune (product) → nullALIS (AI core)" alongside "MNDA delayed by Neptune budget" and considered them complementary — which is correct in the static sense but misses that "Neptune as codename" was superseded by "Nullalis as codename."

**Fix shape**: include `updated_at` deltas in the judge prompt + system-prompt nudge ("if a newer entry contradicts an older one, flag the older as loser"). Or run the judge in two passes: (1) cluster, (2) temporal-rank within cluster. ~50-100 LOC + careful prompt engineering.
