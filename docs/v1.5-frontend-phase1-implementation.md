### 1.5 — API fetch wrappers

export async function fetchBrainGraph(
  userId: string,
  opts?: { since?: number; max_nodes?: number; node_kinds?: string }
): Promise<BrainGraphResponse> {
  const params = new URLSearchParams();
  if (opts?.since !== undefined) params.set("since", String(opts.since));
  if (opts?.max_nodes !== undefined) params.set("max_nodes", String(opts.max_nodes));
  if (opts?.node_kinds) params.set("node_kinds", opts.node_kinds);
  const qs = params.toString();
  const url = `/api/v1/users/${userId}/brain/graph${qs ? `?${qs}` : ""}`;
  const response = await fetch(buildApiUrl(url), {
    headers: authHeaders(),
  });
  if (!response.ok) throw new Error(`brain/graph ${response.status}`);
  return response.json();
}

### Corrections: errors in the playbook body

**A1 — `authHeaders()` does not exist. Use `backendAuthRequest()` for `/api/agent/*` BFF routes.**

WS1.5 originally showed `fetch(buildApiUrl(url), { headers: authHeaders() })` — this will not compile. There is no `authHeaders` export in `src/lib/api.ts`. The codebase has two callers depending on which layer you're talking to:

- `apiRequest(path, options?)` — primary path, hits `getApiBase()` (the express BFF)
- `backendAuthRequest(path, options?)` — same destination but through `backendRequest`, used for the canonical agent surface

**For the brain endpoints specifically, use `backendAuthRequest` against `/api/agent/brain/*`.** Frontend MUST NOT call nullalis paths (`/api/v1/users/{id}/brain/*`) directly — those exist for documentation only. The express BFF mounts `/api/agent/brain/{graph,timeline,compose}` and proxies to nullalis with the canonical bigint user_id derived from auth context. See zaki-prod `backend/src/index.js` (search for `/api/agent/brain/`) and the `Routing convention` addendum at the top of `docs/frontend-vision-brief.md`.

```typescript
// Correct pattern — mirrors fetchAgentSessionHistory + fetchAgentMe
export async function fetchBrainGraph(
  _userId: string,
  opts?: { since?: number; max_nodes?: number; node_kinds?: string }
): Promise<BrainGraphResponse> {
  const params = new URLSearchParams();
  if (opts?.since !== undefined) params.set("since", String(opts.since));
  if (opts?.max_nodes !== undefined) params.set("max_nodes", String(opts.max_nodes));
  if (opts?.node_kinds) params.set("node_kinds", opts.node_kinds);
  const qs = params.toString();
  const response = await backendAuthRequest(
    `/api/agent/brain/graph${qs ? `?${qs}` : ""}`
  );
  if (!response.ok) throw new Error(`brain/graph ${response.status}`);
  return response.json() as Promise<BrainGraphResponse>;
}
```

Same pattern for `fetchBrainTimeline` and `postBrainCompose`. The `userId` argument is retained on the function signature for query-key stability but is no longer interpolated into the URL — auth context derives it.

> **Why `_userId` not removed entirely:** call sites in `BrainPage`, `useBrainGraph`, `useBrainTimeline`, `useBrainCompose` already pass userId through. Keeping the signature intact means TanStack Query keys (`["brain", "graph", userId, ...]`) stay stable across this fix without a fan-out refactor.

**A8 — Timeline hook needs `useInfiniteQuery`, not `useQuery`.**

`useSpaces.ts` uses basic `useQuery` — do not mirror it for Timeline. Use TanStack's `useInfiniteQuery`:

```typescript
import { useInfiniteQuery } from "@tanstack/react-query";

export function useBrainTimeline(userId: string) {
  return useInfiniteQuery({
    queryKey: ["brain", "timeline", userId],
    queryFn: ({ pageParam }) => fetchBrainTimeline(userId, { cursor: pageParam as string | undefined }),
    initialPageParam: undefined as string | undefined,
    getNextPageParam: (lastPage) => lastPage.next_cursor ?? undefined,
    enabled: !!userId,
  });
}
```
