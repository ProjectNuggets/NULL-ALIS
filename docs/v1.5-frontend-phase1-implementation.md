## zaki-prod conventions you MUST follow

- **Stack:** Vite + React 18 + TypeScript + Tailwind + Lucide icons
- **i18n:** en + ar (RTL aware) — every user-facing string goes through `useTranslation()` and `i18n/` files
- **Auth:** `TOKEN_KEY = "zaki.auth.token"` in localStorage; `buildApiUrl()` in `src/lib/api.ts`; `getConfiguredApiBase()` in `src/lib/runtimeEnv.ts`
- **API client extension:** **extend `src/lib/api.ts`** — DO NOT create a new `src/lib/api/` directory. The brief was wrong about this.
- **Components live at** `src/app/components/{chat,memory,sidebar,agent,...}` — pick the matching dir
- **State:** Zustand stores in `src/stores/`
- **Queries:** TanStack Query hooks in `src/queries/`
- **Tests:** Jest unit tests; Playwright e2e in `e2e/`
- **Dark mode:** already wired via CSS vars + Tailwind `dark:` modifier
- **Mobile:** mobile-first; existing `MobileSidebar.tsx` + `MobileHeader.tsx` patterns

**Design language reference:** **Claude Desktop app**. Specifically:
- Minimal sidebar, conversation-first hierarchy
- Subtle dividers, generous whitespace
- Clean typography (system stack, no decorative fonts)
- Hover states are **subtle**, not bold
- Settings/preferences in **sheets**, not modals (look at `ZakiSettingsSheet.tsx` for the pattern)
- Dark mode is polished, not just inverted
- Loading states use **skeleton placeholders** + the existing `TypingText` / `StreamingIndicator` components

### 1.5 — API fetch wrappers

Append to `src/lib/api.ts`:

```typescript
export async function fetchBrainGraph(
  userId: string,
  opts?: { since?: number; max_nodes?: number; node_kinds?: string }
): Promise<BrainGraphResponse> {
  const params = new URLSearchParams();
  if (opts?.since !== undefined) params.set("since", String(opts.since));
  if (opts?.max_nodes !== undefined) params.set("max_nodes", String(opts.max_nodes));
  if (opts?.node_kinds) params.set("node_kinds", opts.node_kinds);
  const qs = params.toString();
  const url = `/api/v1/users/${userId}/brain/graph${qs ? `?${qs}` : ""}`;
  const response = await fetch(buildApiUrl(url), {
    headers: authHeaders(),
  });
  if (!response.ok) throw new Error(`brain/graph ${response.status}`);
  return response.json();
}

// Same shape for fetchBrainTimeline + postBrainCompose. Copy the curl

// shapes.
```

## Workstream 2: Compaction visualization

Backend exposes `context_pressure_percent` on `GET /api/v1/users/:id/sessions/:key/context`. Frontend doesn't render it.

### 2.1 — Add to `NullalisRuntimeWidgets.tsx`

Just below the approval card (or in its own widget), show a context-pressure ring/bar:

```tsx
// 0-50: green; 50-75: yellow; 75-90: orange; 90+: red
function ContextPressureRing({ percent }: { percent: number }) {
  const color = percent < 50 ? "green" : percent < 75 ? "yellow" : percent < 90 ? "orange" : "red";
  return (
    <div className="flex items-center gap-2 text-xs">
      <div className={`w-2 h-2 rounded-full bg-${color}-500`} />
      <span>{percent}% context used</span>
    </div>
  );
}
```

Position: in the message composer footer (subtle), or as a status pill in the chat header.

Backend already emits compaction events on the SSE stream. Look for `compaction.fired` or similar event type. **ASK NOVA** for the exact event name if unclear — backend logs are the source of truth.

## Workstream 5: Unlock-value (V1.5 hidden capabilities)

### 5.1 — Brain compose flow on graph view

When user multi-selects 2+ nodes in `BrainGraphView`, show a "Synthesize selected" button that opens `BrainComposeModal`.

Modal shape:
- Auto-suggested title (concatenation of source titles)
- Editable title field
- Editable content textarea (V1.5 caller provides; V1.6 will auto-fill via LLM trigger)
- Submit → POST /brain/compose
- On success: animate the new node into the graph with reference edges to selected sources

### 5.2 — "Synthesized" badge on compose memories

On graph + timeline, nodes/entries with metadata `synthesized_by: "agent" | "user"` get a small badge:
- agent-authored: blue "AI synthesized"
- user-authored: purple "You synthesized"

Note: V1.5 graph response doesn't expose `synthesized_by` directly on the node; you'd need to call /brain/compose first or read metadata. **For V1.5: skip the badge until V1.6 exposes synthesized_by on graph nodes.** Document the gap; render anyway when /brain/compose response is received during the same session.

### 5.3 — `valid_to` deprecated visual reservation

Reserve a "deprecated" visual treatment (faded color, strikethrough, or "superseded" badge) for nodes/entries with non-null `valid_to`. V1.5 always returns null; V1.6 corrections will trigger this. **Wireframe the visual now so V1.6 doesn't need redesign.**

### 5.4 — ASK NOVA checkpoint

- Compose modal: does it open in-place over the graph, or as a sheet?
- Synthesis text: V1.5 manual entry — should we suggest a starter template (e.g. "Across these N memories, ZAKI sees that...")?
