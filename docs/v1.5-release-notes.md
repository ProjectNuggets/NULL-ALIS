# V1.5 release notes

**Ship target:** 2026-05-05 (alongside V1)
**Codename:** the second brain
**Author:** backend (Claude + Nova)

---

## What V1.5 ships

V1.5 turns ZAKI's memory from a black box into a **legible, navigable, user-trustable surface**. Three ideas land together:

1. **Bi-temporal memory** — corrections don't erase history; they invalidate via `valid_to`. Every fact is auditable across time.
2. **The brain page** — three new endpoints (`/brain/graph`, `/brain/timeline`, `/brain/compose`) power a frontend `/brain` page where users SEE their memory as a navigable structure.
3. **Provenance-aware synthesis** — both the agent (`compose_memory` tool) and the user (`/brain/compose` endpoint) can fold connected memories into one consolidated fact with visible authorship + references.

This is the V1.5 differentiation: most agent products have a black-box memory. ZAKI's is legible. **Trust comes from visibility.**

---

## What you get on the user side

### `/brain` page (frontend ships separately)

Three views, locked JSON contracts, ready for Codex+Opus implementation against the current zaki-prod design system:

- **Graph** — nodes (memories) + edges (session chain, semantic similarity via pgvector cosine, reference links from compose). Up to 500 nodes, capped + ranked, with `semantic_degraded` flag for graceful degrade when pgvector hiccups.
- **Timeline** — chronological list, cursor-paginated (stable across concurrent writes), with optional date-range + session-filter (V1.6) parameters.
- **Compose** — multi-select 2+ nodes, submit synthesis text, server validates references exist, lands a new memory with provenance metadata. Distinct `synthesized_by:"user"` so frontend renders different visual.

### Conversational habits

- Agent now mentions `/brain` when relevant ("you can see your memory there", "the synthesis is now visible on /brain"), making the legibility a habit not a hidden feature.
- Agent routinely consolidates connected memories via `compose_memory create`. Sources stay visible alongside the synthesis (audit trail).

## What you get on the backend side

### Schema (one column, zero migrations)

`memories.valid_to BIGINT` — nullable, partial-indexed. V1.5 always writes NULL (every memory is "always valid"). V1.6 correction classifier populates with `extract(epoch from now())::bigint` to retire facts. Bi-temporal pattern lifted from Graphiti (Apache-2.0).

The classifier in V1.6 will:
- Read `event_type='traversal'` rows for user-attention learning signal
- Read `event_type='compose'` rows for synthesis-decision signal
- Write `event_type='memory_decision'` rows back (Mem0 namespace, locked V1.5 day-2)
- Set `valid_to=now()` to retire memories instead of deleting

**Zero V1.6 schema work needed.** The substrate is the V1.5 feature.

### New endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/v1/users/{id}/brain/graph` | GET | Nodes + edges (session/semantic/reference) |
| `/api/v1/users/{id}/brain/timeline` | GET | Cursor-paginated chronological list |
| `/api/v1/users/{id}/brain/compose` | POST | User-driven synthesis with server-validated references |

All three:
- Tenant + user-scoped (mirror `/sessions` auth)
- Bi-temporal correctness inherited (superseded entries filtered server-side at storage layer)
- UTF-8-safe summary truncation (no codepoint splits for Arabic / Chinese / emoji content)
- Comprehensive 400 error shapes for client-side surfacing
- Traversal-event logging on graph + timeline access (V1.6 classifier substrate)

### New agent tool

| Tool | Purpose |
|---|---|
| `compose_memory create` | Agent-driven synthesis. Required when 2+ memories cluster around the same fact. References stored in metadata; content stays pure. |

Agent prompt teaches the trigger discipline (compose for clustering, NOT for single-source rewrites; sources stay visible).

Tool catalog count: 30 → 31 (extras 34 → 35).

### New memory primitives

- `Memory.storeWithMetadata` — vtable extension (optional slot, defaults null → backward-compat). Production zaki_postgres engine implements; other engines fall back to plain `store` with metadata silently dropped.
- `state_mgr.upsertMemoryWithMetadata` — postgres write path with JSONB metadata + `compose` audit-event side effect.
- `state_mgr.listMemoriesMetadata` — batch JSONB read for /brain/graph reference-edge enrichment. Single SQL query; no N+1.
- `state_mgr.existsMemoryKeys` — batch existence check for server-side reference validation.
- `state_mgr.insertTraversalEvent` — V1.6 classifier substrate. Logs `traversal` rows when /brain pages are viewed.

### Vector store extension

- `VectorStore.pairwiseSimilarities` — single SQL self-join for /brain/graph semantic edges. ivfflat-indexed; bounded by max_pairs param. Optional vtable slot — graceful degrade to empty slice when not implemented.
- `EdgeResult` struct + `freeEdgeResults` helper.

## What V1.5 explicitly does NOT include

These are **V1.6 work**, intentionally scoped out of V1.5:

1. **LLM-driven `/brain/compose`** — V1.5 caller provides synthesis text. V1.6 will accept just `{references[]}` and trigger an agent turn to generate the synthesis.
2. **ADD/UPDATE/DELETE classifier** — the substrate lands V1.5 (bi-temporal schema, traversal-event log, Mem0 namespace). The classifier itself ships V1.6.
3. **Source retirement on compose** — V1.5 keeps source memories visible. V1.6 will allow `retire_sources: true` to set `valid_to=now()` on sources fully replaced by the synthesis.
4. **Recall-path event logging** — V1.5 logs only /brain page views (low-volume, user-driven). V1.6 adds sampled recall-path logging.
5. **Filter-aware reads on non-SQL engines** — markdown / lru / redis / lancedb / lucid carry the `valid_to` field via struct passthrough but don't filter on read. V1.5 invisible (always-null path); V1.6 closes when classifier populates valid_to.
6. **Source-existence validation in compose_memory tool** — tool path trusts the agent. Endpoint path validates via existsMemoryKeys.
7. **Auto-suggested compose candidates** — V1.6 will show "these N nodes look like they cluster, synthesize?" on /brain page.
8. **session_filter param on /brain/timeline** — V1.6.

## Known acceptable gaps for V1.5 ship

1. **`/brain/compose` thin endpoint test coverage** (only 405 + 400 edge cases). Manual smoke tested against live PG via the integration test at `27b0d5f`. Acceptable.
2. **No live integration test for pgvector `pairwiseSimilarities`** (requires populated `memory_embeddings_e5_1024` table). Verified via SQL inspection + graceful-degrade tests. Recommended: run a `curl /brain/graph` against staging post-deploy and eyeball the response.
3. **`memory_events` table grows without bound.** V1.6 retention policy queued.
4. **Two pre-existing test failures** in `bootstrap.integration_test.test.factory creates NullBootstrapProvider for memory backend` and `zaki_state.test.postgres_pool_releases_on_exec_error` (intentional error logging). Predate V1.5 by 500+ commits. Test infrastructure quirks, not code bugs. Track separately.
