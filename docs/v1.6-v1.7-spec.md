## 1. The headline

**Today our memory has S-tier substrate and B-tier extraction.** Reads are at parity with the references (RRF + MMR + temporal decay + optional LLM rerank, already shipped in `RetrievalEngine`). Writes are not — there is no per-turn LLM extraction pass, no typed edges in storage, no conflict detection, and the bi-temporal model is one field deep. The /brain page is rendering what we have. What we have is shallow.

**V1.6 closes the write-side gap to Mem0+Graphiti parity.** V1.7 adds depth (entity vector plane, communities, profile materialization, FE contract upgrade). After both ship, nullalis is at full parity with the open-source SOTA.

**Total scope:** 12 V1.6 seams + 8 V1.7 seams, code-grounded with file:line per audit §E. Every prompt is a verbatim port from a reference repo's source, not a marketing page. No new architecture invented — adopting battle-tested patterns from Mem0 V3 and Graphiti.

### 3.1 — Schema migration: V1.6 columns + memory_entities table

**Single DDL block** appended to `src/zaki_state.zig::ensureMemorySchema` after the current `valid_to` ALTER (`:950`). Idempotent, no row rewrites, online-safe.

```sql
ALTER TABLE {schema}.memories
  ADD COLUMN IF NOT EXISTS subject         TEXT,
  ADD COLUMN IF NOT EXISTS predicate       TEXT,
  ADD COLUMN IF NOT EXISTS object_key      TEXT,
  ADD COLUMN IF NOT EXISTS link_type       TEXT,            -- 'updates' | 'extends' | 'derives' | 'contradicts'
  ADD COLUMN IF NOT EXISTS valid_at        BIGINT,          -- event time, when the fact became true
  ADD COLUMN IF NOT EXISTS invalid_at      BIGINT,          -- event time, when the fact stopped
  ADD COLUMN IF NOT EXISTS expired_at      BIGINT,          -- system time of close-out
  ADD COLUMN IF NOT EXISTS reference_time  BIGINT,
  ADD COLUMN IF NOT EXISTS episodes        TEXT[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS attributed_to   TEXT,            -- 'user' | 'assistant' (Mem0)
  ADD COLUMN IF NOT EXISTS lemmatized      TEXT,
  ADD COLUMN IF NOT EXISTS attribution     TEXT,            -- 'agent_tool' | 'extraction_classifier' | 'compose' (D4 mitigation)
  ADD COLUMN IF NOT EXISTS is_latest       BOOLEAN DEFAULT TRUE,
  ADD COLUMN IF NOT EXISTS parent_memory_id TEXT;

CREATE INDEX IF NOT EXISTS idx_memories_subject     ON {schema}.memories(user_id, subject)     WHERE subject IS NOT NULL;
CREATE INDEX IF NOT EXISTS idx_memories_object_key  ON {schema}.memories(user_id, object_key)  WHERE object_key IS NOT NULL;
CREATE INDEX IF NOT EXISTS idx_memories_parent      ON {schema}.memories(user_id, parent_memory_id) WHERE parent_memory_id IS NOT NULL;
CREATE INDEX IF NOT EXISTS idx_memories_lemmatized  ON {schema}.memories USING gin (to_tsvector('simple', lemmatized));

CREATE TABLE IF NOT EXISTS {schema}.memory_entities (
    id              TEXT PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES {schema}.users(user_id) ON DELETE CASCADE,
    name            TEXT NOT NULL,
    name_lower      TEXT NOT NULL,
    entity_type     TEXT NOT NULL DEFAULT 'PROPER',  -- PROPER | QUOTED | COMPOUND | NOUN
    name_embedding  VECTOR,                           -- pgvector, separate plane from memory_vectors
    linked_memory_ids TEXT[] DEFAULT '{}',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, name_lower)
);
CREATE INDEX IF NOT EXISTS idx_entities_user ON {schema}.memory_entities (user_id);
CREATE INDEX IF NOT EXISTS idx_entities_vec  ON {schema}.memory_entities USING ivfflat (name_embedding vector_cosine_ops);
```

**Why a separate entity vector plane:** mixing entity-name embeddings (typically 1-5 words) with memory-content embeddings (typically 50-200 words) cross-contaminates the cosine ANN. Entity NER queries return random memory chunks; memory content queries return random entity stubs. Mem0 and Graphiti both keep these separate — we follow.

**MemoryEntry struct extension:** mirror the new columns in `src/memory/root.zig:271-330`. Backward-compat: all new fields nullable / defaulted.

**Brain-hygiene update:** `BRAIN_HIDDEN_PREFIXES` in `src/memory/root.zig` doesn't need to change — entity rows live in a different table; new memory keys still flow through the same prefix taxonomy.

**Acceptance:**
- `zig build test -Dengines=all` green with NULLALIS_POSTGRES_TEST_URL set; the existing schema test confirms migration applies cleanly to a populated table
- New PG smoke test: `V1.6 schema migration applies cleanly + memory_entities round-trips a row`

**Estimated:** 1 day. Atomic commit.

### 3.3 — Per-turn LLM extraction classifier

**New file:** `src/agent/extraction.zig`. Public surface:

```zig
pub const ExtractedMemory = struct {
    text: []const u8,                                 // 15-80 word atomic fact
    attributed_to: enum { user, assistant },
    linked_memory_ids: []const []const u8,            // existing keys this memory references
    link_type: ?enum { updates, extends, derives, contradicts } = null,
    subject: ?[]const u8 = null,
    predicate: ?[]const u8 = null,
    object_key: ?[]const u8 = null,
};

pub const ExtractionResult = struct {
    memories: []ExtractedMemory,
    skipped_dedup_md5: usize,
    skipped_dedup_cosine: usize,
    llm_call_ms: u64,
    estimated_tokens: u64,
};

pub fn runExtraction(
    allocator: std.mem.Allocator,
    llm_client: *llm.Client,
    rt: *MemoryRuntime,
    user_id: i64,
    session_id: ?[]const u8,
    user_message: []const u8,
    assistant_reply: []const u8,
    last_k_messages: []const MessageEntry,            // cap at 10
) !ExtractionResult;
```

**Internals (Mem0 V3 + nullalis-specific):**

1. `_build_existing_memories` — `rt.search(user_message, top_k=10, session_id)` to populate the classifier's existing-memory context. Mem0 default top_k=10. Apply the 300-char-per-message truncation per Mem0's `PAST_MESSAGE_TRUNCATION_LIMIT`.
2. `_remap_uuids_to_indices` — anti-hallucination remap: pass existing memories as `{"id":"0","text":...}, {"id":"1",...}` to the LLM, then reverse-map the response (gap #25 from audit).
3. `_build_extraction_prompt` — port `ADDITIVE_EXTRACTION_PROMPT` verbatim from `mem0_spec.md` §3b. Apply `AGENT_CONTEXT_SUFFIX` when scope is agent-only (V-infinity Pillar 2 alignment).
4. `_call_llm_extraction` — single LLM call, `response_format = json_object`, model = D1 choice (Kimi K2 default). Off-thread spawn matching `MemoryRuntime.warmupSession` pattern at `src/memory/root.zig:1311`.
5. `_md5_pre_filter` — `std.crypto.hash.Md5` over normalized text; check against existing rows' `content_hash` column (already populated, gap #13 wires it as Mem0-style filter).
6. `_cosine_dedupe_recent` (D4 mitigation) — for each candidate, embed → cosine vs recent rows in same session → skip if > 0.92.
7. `_persist_extracted` — write each survivor via new `state_mgr.upsertMemoryV2(allocator, user_id, key, content, full_v16_metadata)`. Emits a `memory_events` row with `event_type='extraction'` (new event_type alongside existing `upsert`/`compose`/`traversal`).

**Invocation point:** `src/agent/root.zig:~3175`, after `rt.drainOutbox`, off-thread:

```zig
    extraction.runExtraction(...) catch |err| {

```

### 3.4 — Conflict detection + bi-temporal close-out (Graphiti port)

**New file:** `src/agent/edge_resolution.zig`. Functions:

```zig
pub const Contradiction = struct {
    existing_key: []const u8,
    invalid_at: i64,                                  // event time
    expired_at: i64,                                  // system time
    rationale: []const u8,                            // LLM-emitted reason
};

pub const DedupeContradictionResult = struct {
    duplicate_indices: []const usize,                 // input indices that match existing memories (skip write)
    contradictions: []const Contradiction,            // existing memories to close out
};

pub fn resolveContradictions(
    allocator: std.mem.Allocator,
    llm_client: *llm.Client,
    rt: *MemoryRuntime,
    user_id: i64,
    new_memories: []const ExtractedMemory,
    existing_candidates: []const MemoryEntry,         // related-by-endpoint hybrid search
    invalidation_candidates: []const MemoryEntry,     // broader hybrid neighborhood
) !DedupeContradictionResult;
```

**Implementation:** port verbatim from `graphiti_spec.md` §4.1:
- Single LLM call merging `EXISTING_FACTS + INVALIDATION_CANDIDATES` into one prompt
- Output schema: `{duplicate_facts: int[], contradicted_facts: int[]}`
- Continuous-idx trick: pre-fetched lists are concatenated and indexed 0..N-1; LLM emits indices, we map back to keys

**Three close-out paths** (port from graphiti_spec.md §2.5):
- (A) explicit `invalid_at` from extraction LLM → write directly
- (B) newer fact contradicts older edge → close older with `invalid_at = new.valid_at`, `expired_at = now()`
- (C) older fact arrives after newer one → new memory born expired (`invalid_at = newer.valid_at`)

**New writer:** `state_mgr.setMemoryInvalidation(user_id, key, valid_to, invalid_at, expired_at)` — single SQL UPDATE. Insertion: `src/zaki_state.zig` near `forgetMemory` at `:2541`. Also bumps `is_latest = false` on the closed-out row.

**Invocation:** end of `extraction.zig::_persist_extracted` — after MD5/cosine pre-filter, before write, run dedupe+contradiction LLM judge; skip duplicates, write contradictions to invalidation list, persist remainder.

**Acceptance:** unit test feeding a 3-memory existing set + 1 contradicting new memory yields 1 contradiction with correct `(invalid_at, expired_at)` math. PG smoke test confirms the closed-out row's `valid_to` is past, `MEMORIES_VALIDITY_FILTER` hides it from agent retrieval, agent context now sees only the new memory.

**Estimated:** 2 days. Atomic commit.

## 5. Risks + mitigations summary

| # | Risk | Mitigation |
|---|---|---|
| 1 | Per-turn LLM cost compounds | Off-thread + cosine pre-filter + shadow A/B before flip |
| 2 | Schema migration on populated DB locks | All `ADD COLUMN IF NOT EXISTS` are metadata-only, instant. GIN index on lemmatized uses `CREATE INDEX CONCURRENTLY` outside transaction |
| 3 | Backfill is expensive ($37/user one-shot) | Hybrid C: lemmatize cheap-SQL on migration, extract lazily on retrieval miss |
| 4 | Two writer paths (R14 + extraction) duplicate | D4 mitigation: cosine-dedup at write + `attribution` column for future reconciliation |
| 5 | Entity vector table cross-contaminates ANN | Separate `memory_entities` table, parallel pgvector plane |
| 6 | Extraction prompt drift across providers (Kimi vs OpenAI) | Provider-specific prompt variants; test corpus pinned per provider |
| 7 | LLM-judge contradiction false-positives close-out healthy memories | Conservative threshold; add `confidence_score` column; manual surfacing on the brain page (V1.7) |
| 8 | Connection pool pressure with 2-3 vector planes | Bump `PgvectorConfig.pool_max` from 4 → 8; share pool across planes |
| 9 | FE contract upgrade breaks Phase-1 work | New `/brain/documents` is additive, doesn't touch existing `/brain/graph` |
| 10 | Shadow A/B inconclusive | Lengthen window or pivot prompt — ship gates on quality, not calendar |
