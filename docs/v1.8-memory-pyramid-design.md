# V1.8 Memory Pyramid Design — first principles, next-generation

**Authored:** 2026-05-04 (pre-compaction). Synthesizes Phase 1 audit (6 confirmed gaps + 5 closed open questions) + Phase 2 research (Mem0/Letta/Anthropic/LangMem/Cognee/Zep). Designed under the locked **next-generation principle** ([[feedback_next_generation]]).

**Status:** Design document. NOT a build plan. Phase 4 maps this onto reality, Phase 5 picks V1.8 sprint, Phase 6 plans, Phase 7 builds.

**Constraint policy:** Where today's implementation forces a compromise, document the compromise + its expiry. Don't bake constraints into the design.

## What the 6 gaps reveal STRUCTURALLY

Looking past individual fixes, the gaps share a pattern:

> **The memory architecture has the right SHAPE but wrong CADENCE.**

Every layer exists. Every primitive is in code. The problems are:
- **Triggers fire at the wrong frequency** (Pass C trigger at 90% of context window — too rare)
- **Pipelines don't compose** (extraction_persist writes memories but not embeddings)
- **Prompts are too permissive** (LLM allowed to skip JSON tail)
- **Wiring TODOs were left open** (judge=null in gateway)
- **Cascades are incomplete** (forget memories without forgetting their edges)

The next-generation principle says: don't build new layers, **make the existing ones reliable, cheap, and composed by default**.

## Proposed framing — not Nova's 4 layers, but 4 PROCESSES + 9 SURFACES

Nova's 4-layer framing (.md / Postgres / vectors / brain) is a **storage** view. It works for "where is data?" but doesn't answer "WHEN does data move?" or "WHAT decides?"

### The framing I propose: 4 processes × 9 surfaces

**4 Processes** (the verbs — when does data move?):

```
P1  CAPTURE      — every message, every tool call, every observation
                   becomes immutable raw data. Cheap. Synchronous.
                   Never blocks the user.

P2  DISTILL      — multi-tier extraction. Cheap deterministic pass
                   on EVERY message (entities, dates, key=value,
                   declarative shapes). LLM extraction at session
                   boundaries OR pressure threshold OR explicit ask.
                   ASYNCHRONOUS. Does not block hot path.

P3  RECONCILE    — contradiction resolution + supersession. Bi-temporal
                   close-out. Sleep-time worker triggered when distill
                   produces a candidate that semantically collides
                   with an existing fact.

P4  REFLECT      — periodic synthesis. Daily summary, weekly synthesis,
                   community-name refresh, decay sweep. Background
                   scheduler. Never on hot path.
```

**9 Surfaces** (the nouns — where does data live?):

```
S0  Stream         (messages table)         — append-only L0; the
                                              cold ground truth
S1  Working set    (in-RAM history)         — current turn's view;
                                              ephemeral
S2  Atomic facts   (memories table)         — extracted/observed/
                                              composed facts; bi-temporal
S3  Vectors        (memory_embeddings_e5)   — pgvector index over S2;
                                              hybrid retrieval surface
S4  Entities       (memory_entities)        — named-entity coreference
                                              graph; 0.95 cosine
S5  Edges          (memory_edges)           — typed predicates between
                                              S2 and S4; bi-temporal
S6  Communities    (memory_communities)     — periodic LPA over S5;
                                              auto-named clusters
S7  Episodes       (summary_*, timeline_*)  — session-boundary prose;
                                              bridges S0→S2→S5
S8  Identity       (durable_fact, profile)  — the user's declared
                                              core: name, preferences,
                                              roles. Highest-priority
                                              warm context.
S9  Markdown       (workspace/MEMORY.md)    — human-readable mirror
                                              of S2 (subset). Read by
                                              users, ALSO read by agent
                                              as fallback substrate
                                              (T11 audit proved this).
```

The processes WRITE TO surfaces; the surfaces FEED retrievers + agents:

```
P1 CAPTURE   →  S0 Stream
                S1 Working set

P2 DISTILL   →  S2 Atomic facts
                S3 Vectors          (every S2 row gets one)
                S4 Entities         (named-entity coref)
                S5 Edges            (typed triples)
                S8 Identity        (high-confidence durable subset)

P3 RECONCILE →  S2 (close-out via valid_to)
                S5 (close-out via is_latest=false)

P4 REFLECT   →  S6 Communities
                S7 Episodes
                S9 Markdown export
```

This framing answers questions Nova's 4-layer framing can't:
- **"When does data move?"** → 4 processes, each with explicit triggers
- **"What's the cost gradient?"** → cheap deterministic > LLM extraction > LLM reflection
- **"What can go wrong?"** → process-by-process failure modes
- **"How do we audit?"** → memory_events records every process invocation

## Process design (the WHEN + WHAT)

### P1 — CAPTURE

**Trigger**: every user message, every assistant message, every tool call result. **Synchronous. Never blocks.**

**Writes**:
- `messages` table (already does this)
- Optional: pre-LLM `autosave_user_*` snapshot (already does)
- Optional: post-LLM `autosave_assistant_*` snapshot (already does)

**Cost**: O(1) per turn. No LLM calls. No embeddings.

**Failure mode**: PG write failure → log + reply still ships (autosaves are best-effort).

**Status**: ✅ WORKING in production today.

### P2 — DISTILL

**Trigger model — multi-tier**:

| Tier | Trigger | Cost | When |
|---|---|---|---|
| **2a Cheap deterministic** | Every user message | μs | Synchronous, post-capture |
| **2b LLM atomic-fact** | Session boundary OR explicit `memory_store(triple)` agent call | ~1 LLM call | Async, post-reply |
| **2c LLM Pass C compaction** | Token pressure ≥ 90% of effective context | ~1 LLM call | Synchronous (acceptable: rare) |

**Tier 2a (NEW — doesn't exist today)**: deterministic cheap pass extracts:
- Named entities (capitalized phrases, @mentions, URL domains)
- Dates / time references (regex)
- Declarative shapes ("I am X", "I work at X", "I prefer X over Y")
- key=value patterns
- → writes low-confidence S2/S4/S5 entries with `attribution=deterministic`, `confidence=0.5-0.7`

**Tier 2b (EXISTS but doesn't fire reliably)**: lifecycle summarizer post-reply.
- Today: emits prose + OPTIONAL JSON tail (LLM omits it most of the time)
- V1.8 fix: tighten prompt + measure compliance; add adversarial eval
- Gap → tier 2a will FLOOR the edge density even when 2b doesn't fire

**Tier 2c (EXISTS but unreachable in practice)**: compaction Pass C.
- V1.8 audit: re-test on Kimi K2.5 (256K context) which Nova confirms IS reachable in production
- Trigger threshold may need re-tuning per real conversation lengths

**Composition**: 2a runs on EVERY message → consistent baseline. 2b adds higher-confidence triples at session ends. 2c handles the rare super-long-context case. **All three write to the same S2/S3/S5 surfaces.**

**Failure modes**:
- 2a regex misses → 2b might catch it
- 2b LLM omits JSON → 2a already wrote the deterministic version
- 2c never fires → 2a + 2b cover the gap

**Status**: ⚠ ONLY tier 2c exists today, and it's effectively unreachable. Tier 2a missing entirely. Tier 2b exists but unreliable.

### P3 — RECONCILE

**Trigger**: any P2 write of a new S2 fact OR S5 edge → checks for semantic collisions in existing S2/S5 → if hit, fires the contradiction judge.

**Behavior** (the existing code path, just reliably wired):
- Mem0-style ADD/UPDATE/DELETE/NOOP classification
- For UPDATE: write new fact + close old (`valid_to=NOW`, `is_latest=false`)
- For DELETE: close old without new
- For NOOP: skip write (dedup)

**The judge is FALLBACK to the same provider+model as P2** (already designed in `compaction.zig:494`). Just needs the gateway wiring fix at `gateway.zig:1562`.

**Cost**: 1 LLM call per ambiguous semantic-similarity hit (cosine band 0.85-0.95). Cheap when running on sidecar.

**Failure mode**: judge LLM error → log + skip (new fact lands, old stays — graceful degradation, audit-flaggable later).

**Status**: ⚠ PIPELINE EXISTS but `judge_provider=null` in gateway → never fires. **Single-line fix.**

### P4 — REFLECT

**Triggers** (multiple, scheduled):
- Session boundary → S7 Episode write (already exists)
- Daily midnight → S6 community recompute + community-name refresh + S7 daily summary
- Weekly Sunday → S7 weekly synthesis (NEW)
- Decay sweep → mark stale edges for inclusion-weight reduction (V1.7b-5 work)

**All async. Never on hot path.**

**Cost**: bounded per-trigger; cap LLM calls (50/recompute already exists for community naming).

**Status**: ⚠ S7 Episodes exist but no daily/weekly. S6 communities work via manual recompute. Decay not yet wired.

## Surface design (the WHERE)

For each surface: read paths, write paths, retrieval signal contributions, hygiene policy.

### S0 — Stream (messages)
**Schema (today)**: `messages(id, user_id, session_id, role, content, created_at)`. Append-only.
**Reads**: agent's history reload on session restart; transcript_read tool; never used by retrieval directly.
**Writes**: P1 only.
**Retention**: per `feedback_cold_memory_auditability` — **forever**. Transcript IS cold tier. Don't trim.
**Status**: ✅ Working.

### S1 — Working set
**In-RAM only.** Truncated by compaction at hot-path token limits. Not persistent.
**Status**: ✅ Working.

### S2 — Atomic facts (memories)
**Schema (today)**: `memories(id, user_id, key, content, memory_type, valid_to, invalid_at, expired_at, is_latest, metadata, content_hash, ...)`. Bi-temporal.
**Reads**: BM25 lexical (recallMemories), key lookup, content hash dedup.
**Writes**: P2 (all three tiers), P3 close-out, P4 episode write.
**Hygiene**: `BRAIN_USER_KEY_FILTER` excludes bookkeeping (autosave_*, summary_*, etc.) from user-visible surfaces.
**Bi-temporal invariant**: `valid_to NULL OR > NOW` = active; otherwise archived.
**Status**: ✅ Bi-temporal correct. ⚠ Many writes don't trigger embedding (G-C).

### S3 — Vectors (memory_embeddings_e5_1024)
**Schema (today)**: `memory_embeddings_e5_1024(user_id, key, embedding vector(1024), updated_at)`. PRIMARY KEY (user_id, key).
**Reads**: pgvector cosine via MemoryRuntime.search → RRF merge with BM25.
**Writes**: should fire on EVERY S2 write. Today only fires for select paths.
**Failure mode (G-C)**: extraction_persist bypasses syncVectorAfterStore.
**V1.8 fix**: enforce "every S2 row has a vector" as an invariant. Add it to memory_doctor health check.
**Status**: ⚠ Coverage 4/43 = 9% in audit. Should be 100%.

### S4 — Entities (memory_entities)
**Schema (today)**: `memory_entities(id, user_id, name, name_lower, entity_type, vec, created_at, ...)`. id = stable hash of name_lower; vec = pgvector embedding of name.
**Reads**: coref lookup at write time (Mem0-style 0.95 cosine).
**Writes**: P2 when an extracted fact has a subject/object that doesn't already exist.
**Status**: ✅ Working — 9 entities for user 7777, JOINs cleanly to edges.

### S5 — Edges (memory_edges)
**Schema (today)**: typed triples (source_key, target_key, predicate, attribution, confidence, weight, valid_from, valid_to, invalid_at, expired_at, is_latest).
**Reads**: brain_graph tool, /brain/local-graph, /brain/communities pipeline, agent retrieval graph_neighbors.
**Writes**: P2 atomic-fact extraction (fires reliably for explicit triple stores; not from prose), P3 close-out.
**G-NEW**: orphan edges accumulate when forgetMemory deletes the source memory without cascading.
**V1.8 fix**: extend `forgetMemory` to UPDATE memory_edges WHERE source_key OR target_key matches.
**Status**: ⚠ Orphan-edge accumulation; otherwise good.

### S6 — Communities
**Schema (today)**: `memory_communities(user_id, community_id, name, generated_at, member_count, name_source)` + `memories.community_id` column.
**Reads**: /brain/communities, brain_graph(action=communities), warm `<active_communities>` block.
**Writes**: P4 nightly + on-demand recompute.
**Status**: ✅ V1.7a-9 shipped; nothing for V1.8.

### S7 — Episodes
**Surfaces**: summary_latest/{session}, session_summary/{session}/{ts}, timeline_summary/{session}/{ts}, durable_fact/{topic}/{hash}, daily_summary/{date}, weekly_synthesis/{week} (NEW).
**Schema (today)**: lives in S2 with prefix-based namespacing (BRAIN_USER_KEY_FILTER hides them from /brain).
**Reads**: agent's continuity warm context (memory_loader); /memory timeline; /memory search.
**Writes**: P4 (session boundary, daily midnight, weekly Sunday).
**V1.8 GAP**: daily/weekly aggregation doesn't exist. Session-bounded only.
**Status**: ⚠ Session-level OK. Day/week missing.

### S8 — Identity
**The user's CORE: name, preferences, roles, what they're working on, who matters to them.** Today scattered across `durable_fact/behavior/*` + `durable_fact/identity/*` + `extracted_*` triples about subject="<self>".
**Proposed**: explicit identity profile blob — single living document refreshed by P4. Always in warm context. Letta-style memory block.
**V1.8 NEW**: introduce identity profile materialization (V1.7b-3 was this, demoted). Worth promoting to V1.8.
**Status**: ❌ Missing as first-class surface.

### S9 — Markdown mirror
**Schema (today)**: `workspace/<user>/MEMORY.md` + per-key files via MarkdownMemory engine. Write-through mirror via zaki_dual.
**Reads**: workspace bootstrap; **agent fallback** when memory_recall misses (proven by T11 audit).
**Writes**: zaki_dual.store forwards to markdown_impl.store (best-effort).
**The audit proved this is high-leverage**: even with broken vectors, the agent successfully recovered the Helix preference by reading MEMORY.md.
**V1.8 ADOPTION**: lean INTO the markdown surface. Make it ALWAYS up-to-date. Use it as the deterministic reasoning substrate (Anthropic Memory tool pattern).
**Status**: ⚠ Mirror exists but cascade-on-supersession may have gaps. Verify in V1.8.

## Adversarial evals — making the V1.8 fix actually durable

The Phase 2 research was unanimous: **extraction reliability is the universal failure mode**. Mem0 has 97.8% junk after 32 days. Our 0-edges-DB-wide was the same family.

A pure code fix without continuous evaluation will regress. **V1.8 ships with adversarial evals as part of the deliverable**:

```
.audit/v1.8/evals/                                  — committed test suite
  prompt_corpus/                                    — canonical prompts
    identity_writes.txt                             — "My name is X. I work at Y."
    preference_changes.txt                          — supersession scenarios
    multi_entity.txt                                — 3+ entity prompts
    relational_queries.txt                          — should fire brain_graph
    long_context_pass_c.txt                         — Pass C trigger sequence
  expected.json                                     — for each prompt, what
                                                       MUST land per layer
  run_eval.sh                                        — sends prompts, snapshots
                                                       PG, asserts expected,
                                                       reports F1 per gap
```

CI gates: every commit touching extraction or memory paths runs the eval; >5% F1 regression blocks merge.

This is the V1.8 SHIP CONDITION, not a nice-to-have.

## Trade-offs we deliberately accept

| Trade-off | Why |
|---|---|
| Tier 2a cheap pass produces noise (low-confidence facts) | Floor of edge density beats zero. Confidence column lets retrieval down-weight noise. |
| Embedding every S2 row costs ~5ms × N writes | Cheap (pgvector + e5-1024 batched). Beats invisibility. |
| Judge fires on every triple = 1 LLM call per ambiguous candidate | Cap at 50/min via rate limiter. Acceptable. |
| Daily summary requires a background scheduler | Cron via existing tenant-job framework. ~1 day setup. |
| MEMORY.md mirror grows unbounded | Compress old entries to summary blocks at quarter boundaries. V1.9. |
