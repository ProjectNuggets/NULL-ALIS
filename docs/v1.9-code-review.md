## Findings

### V1.9-1 вАФ `cascadeRenameEntity`

| # | Sev | Finding | Disposition |
|---|---|---|---|
| 1 | рЯФі | **Memory leak: `new_id_owned` allocated at zaki_state.zig:4547 via `dupeResultValue(self.allocator, ...)`. Freed in case-only branch (4552) but NOT in success path. Function exit at line 4694 dupes via caller's allocator вЖТ caller-owned copy returned, but the original self.allocator-owned slice is never freed.** Per-call leak: 32 bytes (entity_id hex). Over 1000 calls: ~32KB. Real production bleed. | **V1.9-Rev fix:** add `defer self.allocator.free(new_id_owned)` after line 4547. Remove explicit free at 4552 (let defer handle it). The case-only branch returns early; defer runs at return. |
| 2 | рЯЯ° | Early-return at lines 4514-4520 (entity not found) uses `""` (string literal) for `old_id` / `new_id`, but `CascadeRenameResult` contract says "allocator-owned slice." Saved at runtime by `deinit`'s `len > 0` guard, but contract violation that could surface as drift if the guard is ever removed. | **V1.9-Rev fix:** use `try allocator.alloc(u8, 0)` to return zero-length allocator-owned slices, matching the contract. |
| 3 | рЯФµ | `ON CONFLICT (user_id, source_key, predicate, target_key) WHERE is_latest DO NOTHING` on the INSERT new edges (line 4599). Correct: prevents duplicate edge if the new (subject,predicate,object) triple already existed elsewhere. вЬУ | Working as designed |
| 4 | рЯФµ | `inserted_count` parsed via `PQcmdTuples` text вЖТ integer. Catches the `INSERT 0 N` Postgres tag format. Empty string вЖТ 0. вЬУ | Working as designed |
